# 实施文档

## 1. 环境准备

### 1.1 系统要求
- **操作系统**: Windows 10+ / Linux (Ubuntu 20.04+) / macOS 11+
- **Python**: 3.8 或更高版本
- **内存**: 建议 512MB 以上（PDF大文件时需要更多）
- **磁盘**: 项目本身 <5MB，运行时数据根据标注数量增长

### 1.2 安装步骤

#### Windows 环境

```powershell
# 1. 进入项目目录
cd "e:\Projects\TAB Score Viewer"

# 2. 创建虚拟环境（如尚未创建）
python -m venv venv

# 3. 激活虚拟环境
.\venv\Scripts\activate

# 4. 安装依赖（使用清华源加速）
pip install PyQt5 PyMuPDF Pillow PyGuitarPro -i https://pypi.tuna.tsinghua.edu.cn/simple --trusted-host pypi.tuna.tsinghua.edu.cn

# 5. 运行程序
python "TAB Score Viewer.py"
```

#### 打包为 EXE（可选，无需 Python 环境）

```powershell
# 1. 安装 PyInstaller
pip install pyinstaller -i https://pypi.tuna.tsinghua.edu.cn/simple

# 2. 执行打包（使用预配置的 spec 文件）
pyinstaller guitar_tab_viewer.spec

# 3. 运行打包结果
dist\guitar_tab_viewer\guitar_tab_viewer.exe
```

> **打包说明**:
> - 使用 onedir（目录）模式，输出到 `dist/guitar_tab_viewer/` 文件夹
> - 图标: icon.ico 同时作为 exe 文件图标和运行时窗口/任务栏图标
> - 包含: 翻译文件(locales/)、音频DLL、图标等所有依赖
> - 启动快: 相比单文件模式无需解压，即点即用

## 3. 使用指南

### 3.1 基本操作流程

```
启动程序 → 选择文件夹 → 双击文件打开谱面 → 播放/调速/添加标注
```

### 3.2 详细操作说明

#### 打开文件
1. 点击「选择文件夹」按钮
2. 选择存放吉他谱图片/PDF的文件夹
3. 文件列表会自动过滤显示支持的格式
4. 双击任意文件即可打开

#### 播放控制
1. 点击「▶ 播放」或按空格键开始播放
2. 谱面会自动向上滚动
3. 拖动底部进度条可跳转到任意位置
4. 再次点击或按空格暂停

#### 速度调节
1. 拖动速度滑块调整基础速度（值越小越快）
2. 对于变速练习需求，点击「📊 速度曲线」按钮：
   - 在曲线上点击添加控制点
   - 拖动控制点调整位置和速度
   - 勾选「启用速度曲线」生效
   - 可使用预设模板快速应用常用曲线

#### 循环播放
1. 在循环模式下拉框中选择模式
2. 「区域循环(A-B)」模式：
   - 按住 Ctrl 点击进度条设置起点 A
   - 按住 Shift 点击进度条设置终点 B
   - 进度条上会高亮显示 A-B 区域
   - 播放到 B 点后自动回到 A 点

#### 添加标注
1. **方式一**: 在谱面上双击空白处 → 弹出编辑对话框 → 输入文本和样式 → 确定
2. **方式二**: 点击「📝 标注」按钮打开管理器 → 点击「新增」→ 编辑 → 确定
3. **编辑已有标注**: 双击谱面上的标注文字，或在管理器中选中后点「编辑」
4. **移动标注位置**: 左键按住标注拖动到目标位置(实时预览)，松手自动保存
5. **撤销/重做**: `Ctrl+Z` 撤销 / `Ctrl+Y` 重做（全局统一，画布+管理器共享）
6. **自动保存**: 每次标注操作后自动写入同名 `.anno.json` 文件（与谱子同目录）

#### 导出谱面(含标注)
1. 点击工具栏「💾 导出」按钮
2. 选择保存路径和格式:
   - **PNG图片**: 高清位图(2480×3508px @300dpi)，适合查看/打印/分享
   - **PDF文档**: 矢量质量最佳，适合正式文档
3. 导出内容包含: 谱面图像 + 全部标注(颜色/字体/大小均保留)
4. 内容超过一页A4时自动分多页导出，文件名追加页码 `_p1`, `_p2`...

#### 页码导航（多图/PDF模式）
1. 底部左侧出现页码输入框（格式: `[输入框] / 总页数`）
2. 直接输入页码数字后按回车跳转
3. 滚动时页码输入框自动同步当前可见页

#### 鼠标滚轮
- 在谱面区域使用鼠标滚轮上下翻页
- Ctrl + 滚轮：快速翻页
- Shift + 滚轮：精细移动

### 3.3 标注数据说明

标注文件采用**同名策略**，与谱子存放在同一目录:

- **非GTP/单轨**: `谱子名.anno.json`（例: `晚安北京.png.anno.json`）
- **GTP多轨(分轨)**: `谱子名.t{轨道号}.anno.json`（例: `song.gp4.t0.anno.json`, `song.gp4.t1.anno.json`）
  - 每个音轨独立一个标注文件，切换音轨时自动保存当前轨并加载目标轨
- **旧格式(兼容)**: `data/annotations/{hash}.json`（仅读取，不再写入）
- **加载优先级**: 内存缓存 > 新路径 > 旧路径 > 空列表

#### 标注JSON结构示例
```json
[
  {
    "id": "ann_a1b2c3d4",
    "x": 0.5,
    "y": 0.3,
    "text": "这里用泛音演奏",
    "color": "#F97316",
    "font_size": 14,
    "font_family": "Microsoft YaHei",
    "is_bold": false,
    "background_color": "#00000080"
  }
]
```

字段说明:
- `id`: 唯一标识符
- `x`, `y`: 相对坐标 (0.0-1.0)，(0,0)=左上角, (1,1)=右下角
- `text`: 标注文本内容
- `color`: 文字颜色（十六进制）
- `font_size`: 字体大小（像素）
- `font_family`: 字体名称。新建标注时默认使用平台自适应字体（Windows 为 Microsoft YaHei，Linux 为 Noto Sans CJK SC，macOS 为 PingFang SC），用户可通过编辑标注修改
- `is_bold`: 是否粗体
- `background_color`: 背景色（支持透明度）

### 3.6 语言与主题设置

程序支持多语言界面、内置深色/浅色主题以及用户自定义主题，相关选项已集中到设置面板：

1. 在选择窗口点击右上角的「⚙ 设置」按钮
2. 在弹出的设置对话框中选择：
   - **常规** 标签页：语言、主题、UI 字体
   - **GTP 渲染** 标签页：GTP 渲染字体、弦线间距、品格字号等渲染参数
3. 点击「确定」保存，配置自动写入 `config/settings.json`
4. 主题与 UI 字体切换后会实时刷新所有已打开窗口
5. 语言切换后会提示「部分界面文本将在下次启动时完全更新」
6. GTP 渲染参数修改后需重新打开 GTP 文件才能看到效果
7. 点击「恢复默认」可一键将语言、主题、字体及所有渲染参数重置为出厂默认值

**翻译文件位置**: `locales/` 目录
- `zh_CN.json` - 简体中文翻译
- `en_US.json` - 英文翻译
- `ru_RU.json` - 俄文翻译

**新增语言**: 在 `locales/` 目录下创建新的 `{lang_code}.json` 文件，参照现有文件格式编写翻译内容即可。程序启动时会自动扫描该目录并加载所有可用语言。

**自定义主题 (v2.2.0 新增)**:
- 主题目录: 项目根目录 `themes/`（首次启动时自动创建）
- 支持格式: `.json` 和 `.py`
- 文件结构: 必须同时包含 `name`、`display_name`、`ui`、`gtp` 四个字段，且 `ui` 与 `gtp` 必须为非空字典
  ```json
  {
    "name": "my_theme",
    "display_name": "My Theme",
    "ui": { "bg_primary": "#121212", "text_primary": "#E2E8F0", ... },
    "gtp": { "COLOR_BG": "#1E1E2E", "COLOR_TEXT": "#E2E8F0", ... }
  }
  ```
- Python 格式: 模块级变量 `THEME = {...}`，结构与 JSON 相同
- 程序启动时自动扫描，合法主题加载到设置面板的主题选择中
- 自定义主题会同时影响 UI 配色和 GTP 六线谱渲染配色
- 缺失的颜色键会自动使用内置深色主题默认值填充
- 缺少 `name`、`ui` 或 `gtp` 任意字段，或 `ui`/`gtp` 为空字典，该主题文件会被跳过
- 可参考 `themes/example_custom.json` 示例文件

## 4. 常见问题

### Q1: 程序启动报错 `ModuleNotFoundError`
**原因**: 依赖未安装或虚拟环境未激活
**解决**:
```bash
# 确认已激活虚拟环境（命令行前缀应有 (venv)）
.\venv\Scripts\activate   # Windows
source venv/bin/activate  # Linux

# 重新安装依赖
pip install PyQt5 PyMuPDF Pillow -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### Q2: PDF 文件打不开
**原因**: PyMuPDF 未正确安装或 PDF 文件损坏
**解决**:
```bash
pip uninstall PyMuPDF
pip install PyMuPDF -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### Q3: 中文显示乱码
**原因**: 终端编码问题
**解决**: Windows 下设置环境变量:
```powershell
$env:PYTHONIOENCODING="utf-8"
python guitar_tab_viewer.py
```

### Q4: GP7/GP8 (.gp) 文件音符画错弦线
**原因**: GP7/GP8 的 GPIF XML 中 `<String>` 字段 0=最低音弦(底线)，而 ApolloTab 内部约定 0=最高音弦(顶线)，旧版本未做反转映射导致音符上下错位。
**解决**: v1.0.1 已在 `ApolloTab/parser/gpif_parser.py` 中修复，解析时按音轨实际弦数自动反转弦号。请确保 ApolloTab 版本 >= 1.0.1。

### Q5: WEBP 格式无法加载
**原因**: Pillow 版本过低不支持 WEBP
**解决**:
```bash
pip install Pillow>=10.0 --upgrade
```

### Q6: 大 PDF 文件加载慢
**原因**: PDF 渲染为图片是 CPU 密集型操作
**解决**: 正常现象，异步加载不会阻塞界面。首次加载后会缓存。

## 7. 节拍器功能 (v1.1.4)

### 7.1 功能概述

节拍器功能支持在 GTP 模式和图片/PDF 模式下播放规律的点击声，帮助用户把握演奏节奏。

- **GTP 模式**: 自动读取歌曲的 BPM 和拍号，无需手动设置。
- **图片/PDF 模式**: 用户可手动输入 BPM（20~300）和拍号分数（分子 1~16 / 分母 1~32，默认 4/4）。
- **UI 控制**: 右侧面板「节拍器」分组提供启用开关、音量滑块和全局增益滑块。
- **音色**: 固定使用 GM 木鱼音色（重拍 77 High Woodblock，普通拍 76 Low Woodblock）。
- **全局增益**: 独立的增益滑块（1.0x ~ 5.0x，默认 1.5x），配合 MIDI CC#7 通道音量最大，用于提升木鱼音色在伴奏中的突出程度。

### 7.2 实现架构

```
TAB Score Viewer.py (UI/状态/播放同步)
         │
         ▼
GTPPlayer.set_metronome() / play_metronome_only()
         │
         ▼
ApolloTab/audio/metronome.py
  - MetronomeConfig (配置)
  - MetronomeGenerator (事件生成)
         │
         ▼
ApolloTab/audio/midi_converter.py
  - convert() / convert_all_tracks() 混入节拍器事件
         │
         ▼
ApolloTab/audio/synth_engine.py
  - FluidSynth 统一播放乐曲 + 节拍器事件
```

### 7.3 关键技术点

| 项目 | 说明 |
|------|------|
| 通道 | MIDI 通道 15，避免与旋律轨和鼓轨冲突 |
| 鼓组切换 | 通道 15 开头发送 `CC#0=1`、`CC#32=0`、`Program Change=0` 启用 GM 鼓组 |
| 重拍/普通拍 | 每小节第一拍为 77（High Woodblock），其余为 76（Low Woodblock） |
| 点击时长 | 每个点击声持续约 50ms，随后发送 `note_off` |
| 音量映射 | UI 音量 0~100 映射到 MIDI velocity 0~127；全局增益 1.0x~5.0x 再对 velocity 进行放大 |
| 通道音量 | 节拍器通道额外发送 CC#7=127，使该通道输出达到最大，避免被其他乐器掩盖 |
| 力度基准 | 重拍基础力度 100，普通拍基础力度 70；配合默认增益 1.5x，最大音量下重拍可达 127，普通拍约 105 |
| 同步方式 | 节拍器事件与乐曲事件在同一 tick 时间线上排序，由 SynthEngine 统一驱动 |

### 7.4 使用方式

1. 打开任意吉他谱文件（GTP 或图片/PDF）。
2. 在右侧面板找到「节拍器」分组。
3. 勾选「启用节拍器」，调节音量。
4. 若觉得节拍声被乐器掩盖，拖动「增益」滑块（推荐 2.0x~3.0x）。
5. 图片/PDF 模式下可进一步设置 BPM 和拍号分数（如 4/4、6/8、3/4）。
6. 点击播放按钮即可听到节拍器声音。

### 7.5 注意事项

- 节拍器依赖 FluidSynth 和 SoundFont，若音频引擎初始化失败则无法使用。
- 图片/PDF 模式下 `GTPPlayer` 不加载具体歌曲，仅初始化 `SynthEngine`。
- 切换 GTP 音轨或音频模式后，节拍器设置保持生效。
