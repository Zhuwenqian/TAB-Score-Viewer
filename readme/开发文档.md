# 开发文档

## 1. 项目概述

### 1.1 项目名称
TAB Score Viewer（万能吉他谱查看器）

### 1.2 系统支持
| 平台 | 支持方式 |
|------|----------|
| Windows 10+ | 源码运行 / PyInstaller EXE 打包 |
| Linux | 源码运行 / PyInstaller + DEB 打包 |
| macOS 11+ | 源码运行 / PyInstaller APP 打包 |

### 1.3 技术栈
| 类别 | 技术 | 版本要求 |
|------|------|----------|
| 语言 | Python | 3.8+ |
| GUI框架 | PyQt5 | >=5.15 |
| PDF处理 | PyMuPDF (fitz) | >=1.23 |
| 图片处理 | Pillow (PIL) | >=10.0 |
| GTP解析 | guitarpro (PyGuitarPro) | >=0.11 |

### 1.3 设计模式
- **MVC 分离**: SelectionWindow(视图/控制) + DisplayWidget(视图) + 数据模型(dataclass)
- **观察者模式**: PyQt5 信号槽机制实现组件间通信
- **工厂模式**: Worker 类统一封装异步任务
- **命令模式**: 撤销/重做通过快照栈实现

### 2.2 核心类说明

#### 主入口与选择窗口

| 类名 | 职责 |
|------|------|
| `SelectionWindow` | 主窗口（原 SettingsWindow），文件浏览选择，应用深色/浅色主题，管理 DisplayWindow 生命周期 |

#### 谱面显示窗口

| 类名 | 职责 |
|------|------|
| `DisplayWindow` | 谱面主窗口，管理播放状态、标注数据、速度曲线、循环配置、页码导航 |
| `DisplayWidget` | 谱面画布 QWidget，负责图片渲染(缓存优化)、标注绘制/拖动、鼠标事件处理（双击添加/拖动移动/滚轮） |

#### 应用图标系统

| 函数/属性 | 职责 |
|------|------|
| `get_app_icon()` | 图标加载工具函数，自动适配开发/打包两种模式，返回 QIcon 对象 |
| `_APP_BASE_DIR` | 应用根目录常量（脚本目录），用于定位 icon.ico |
| `ICON_PATH` | icon.ico 完整路径常量 |
| `get_font_families()` | 平台自适应字体工具函数，根据 Windows/Linux/macOS 返回推荐字体族列表 |
| `get_font_family()` | 返回当前平台指定类型的首选字体名称（用于 QFont） |
| `get_font_family_css()` | 返回当前平台指定类型的 CSS font-family 字符串（用于样式表） |

**使用方式**:
```python
# 获取应用图标（自动适配运行模式）
icon = get_app_icon()

# 设置应用全局图标（任务栏 + 默认窗口图标）
app.setWindowIcon(get_app_icon())

# 设置单个窗口图标
self.setWindowIcon(get_app_icon())
```

**原理**: 通过 `sys.frozen` 检测 PyInstaller 打包环境
- 开发模式: 从脚本同目录(`__file__`)读取 icon.ico
- 打包模式: 从 exe 所在目录(`sys.executable`)读取 icon.ico

#### 国际化系统(I18n)

| 类名 | 职责 |
|------|------|
| `I18n` | 国际化翻译管理器（单例模式），从locales/目录加载JSON翻译文件，提供t()方法获取翻译文本，支持语言切换和占位符格式化 |

**使用方式**:
```python
# 获取翻译文本（支持占位符）
text = I18n.t("control_panel.play_btn")           # "▶ 播放" 或 "▶ Play"
text = I18n.t("messages.export_success_msg", fmt="PDF", path="/tmp/test.pdf", count=3, mode="")
# 切换语言
I18n.set_language("en_US")
# 获取可用语言列表
langs = I18n.available_languages()  # [("zh_CN","简体中文"), ("en_US","English")]
```

**翻译文件结构**: `locales/{lang_code}.json`，采用嵌套键命名规范（如 `control_panel.play_btn`）

#### 异步加载系统

| 类名 | 职责 |
|------|------|
| `WorkerSignals` | 统一信号定义类 |
| `LoadFileListWorker` | 异步扫描文件夹获取支持格式文件列表 |
| `LoadContentWorker` | 异步加载文件内容（图片/PDF/GTP） |

#### GTP渲染引擎（gtp_engine 独立库）

| 类/模块 | 职责 |
|---------|------|
| `GTPParser` | 解析器，PyGuitarPro原始数据 → GTPSong中介模型 |
| `TabRenderer` | 渲染器，QPainter绘制六线谱 → QPixmap多页输出 |
| `TabLayoutEngine` | 布局引擎，自动换行/分页/坐标计算 |
| `GTPSong/GTPTrack/GTPMeasure/GTPBeat/GTPNote` | 可扩展数据模型（dataclass） |
| `RenderConfig` | 渲染参数配置（线宽/间距/颜色等，全部可调） |
| `TechniqueType` | 技巧枚举（18种演奏技巧，可扩展） |

**gtp_engine 架构**:
```
.gp文件 → PyGuitarPro(guitarpro库) → 原始Song对象
                                          ↓
                              GTPParser._convert_song()
                                          ↓
                              GTPSong(中介数据模型)
                               ├─ GTPTrack(音轨)
                               │   ├─ 调弦(tuning)
                               │   └─ GTPMeasure[](小节数组)
                               │       ├─ 拍号/重复记号
                               │       └─ GTPBeat[](拍数组)
                               │           ├─ 时值(is_dotted)
                               │           └─ GTPNote[](音符)
                               │               ├─ 弦/品格/MIDI音高
                               │               └─ techniques[](技巧列表)
                                          ↓
                              TabRenderer.render()
                                          ↓
                              TabLayoutEngine.layout()  (坐标计算)
                                          ↓
                              QPainter 绘制 → QPixmap[] (多页图像)
```

### 2.3 数据流

```
用户操作 → SelectionWindow → 选择文件夹
                ↓
        LoadFileListWorker (异步)
                ↓
        _on_files_loaded → 更新文件列表
                ↓
用户双击文件 → on_file_double_clicked → show_display()
                ↓
        LoadContentWorker (异步)
                ↓
        _on_content_loaded → images列表 → DisplayWidget.set_images()
                ↓
        播放定时器(_tick) → current_position递增 → DisplayWidget.update()
```

## 3. 关键模块详解

### 3.1 文件加载模块

**原理**: 使用 QThreadPool 全局线程池执行耗时IO操作，通过自定义信号返回结果。

**执行步骤**:
1. 用户选择文件夹 → 创建 `LoadFileListWorker` 提交到线程池
2. Worker.run() 遍历目录，过滤支持的扩展名
3. 通过 `signals.finished.emit(result)` 返回文件列表
4. 主线程 `_on_files_loaded()` 接收并更新 UI

**关键代码位置**:
- [WorkerSignals](guitar_tab_viewer.py) - 信号定义
- [LoadFileListWorker](guitar_tab_viewer.py) - 文件列表加载
- [LoadContentWorker](guitar_tab_viewer.py) - 内容加载

### 3.2 图片渲染模块

**原理**: 将所有图片按比例缩放到画布宽度，垂直拼接绘制。使用 QPixmap.scaled() 进行高质量缩放。

**坐标系统**:
- X轴: 0 = 左边距(10px), ww-20 = 右边距
- Y轴: base_y = -current_position (负值使内容向上滚动)
- 标注坐标: 相对值 (0.0-1.0)，映射到绝对像素位置

**关键代码**: [DisplayWidget.paintEvent](guitar_tab_viewer.py)

### 3.3 速度曲线模块

**原理**: 基于贝塞尔插值的速度映射。
- X轴: 播放进度百分比 (0-100%)
- Y轴: 速度值 ms (越小越快)
- 播放时根据当前进度在曲线上线性插值获取实际速度

**预设模板**:
| 名称 | 特征 |
|------|------|
| 匀速 | 两端点等高直线 |
| 渐快 | 从慢到快 |
| 渐慢 | 从快到慢 |
| 先慢后快 | U型曲线 |
| 先快后慢 | n型曲线 |

**关键代码**: [SpeedCurveEditor](guitar_tab_viewer.py), [_get_curve_speed](guitar_tab_viewer.py)

#### 3.4 标注系统(导入/导出/拖动/撤销重做)

**原理**: 命令模式 + 全局快照栈 + 同名文件策略 + 鼠标拖动交互。

##### 3.4.0 标注拖动
- **交互**: 左键按住标注 → 拖动移动位置 → 松手保存快照+写入文件
- **光标**: 悬停OpenHandCursor / 拖动ClosedHandCursor / 离开ArrowCursor
- **实时预览**: 拖动过程中直接修改annotation.x/y并repaint，松手才持久化
- **防误触**: 位置变化<0.001时不入撤销栈
- **与双击互斥**: mousePressEvent记录按下，mouseDoubleClickEvent在200ms后触发，Qt自动区分

###### 3.4.1 标注文件路径策略(含分轨)
- **新策略(默认)**: 与源文件同名 + `.anno.json` 后缀，存放在谱子同目录（方便分发/备份）
  - 非GTP/单轨: `谱子名.anno.json`（例: `晚安北京.png.anno.json`）
  - **GTP多轨**: `谱子名.t{轨道号}.anno.json`（例: `song.gp4.t0.anno.json`, `song.gp4.t1.anno.json`）
    - 每个音轨独立一个标注文件，切换音轨时自动加载对应轨道的标注
- **旧策略(兼容)**: `data/annotations/{hash}.json`（仅读取，不再写入）
- **分轨内存缓存**: `_annotations_by_track: Dict[int, List[Annotation]]` 字典
  - 切轨时先存当前轨到字典，再从字典/文件加载目标轨
  - 避免重复文件IO，切轨响应更快

#### 3.4.2 全局撤销/重做栈
- `_undo_stack`: DisplayWindow内置，存储历史状态（每次修改前 deepcopy 当前 annotations 列表）
- `_redo_stack`: 存储被撤销的状态（可被Ctrl+Y恢复）
- 最大深度50层防止内存膨胀
- **统一入口**: 画布双击添加、管理器增删改、全部共享同一个全局栈
- **快捷键**: `Ctrl+Z` 撤销 / `Ctrl+Y` 重做（DisplayWindow.keyPressEvent）
- **委托机制**: AnnotationManagerDialog 的 `_save_snapshot()`/`_undo()`/`_redo()` 均委托给父窗口 DisplayWindow

**关键代码**: [DisplayWindow._anno_save_snapshot/_anno_undo/_anno_redo](guitar_tab_viewer.py), [AnnotationManagerDialog._save_snapshot/_undo/_redo](guitar_tab_viewer.py)

#### 3.4.3 A4导出模块
- **PNG导出**: QPixmap(2480×3508px @300dpi) → QPainter绘制图片+标注 → save("PNG")
- **PDF导出**: QPdfWriter(A4, 300dpi) → QPainter绘制图片+标注 → end()
- **自动分页**: 总内容高度 / A4可用高度 = 页数，超出部分自动分多页
- **坐标映射**: 标注相对坐标(0-1) × 总内容高度 = 导出画布绝对像素位置

**关键代码**: [_export_to_a4](guitar_tab_viewer.py), [_render_to_a4_png](guitar_tab_viewer.py), [_render_to_a4_pdf](guitar_tab_viewer.py), [_draw_annotations_on_canvas](guitar_tab_viewer.py)

#### 3.4.4 播放性能优化
- **图片缩放缓存(`_cached_scaled`)**: DisplayWidget预缓存所有scaled图片，paintEvent直接使用，避免每帧`scaled()`CPU密集操作。resizeEvent/set_images时自动失效重建。
- **UI更新节流(`_tick`)**: 进度条位置每帧更新(轻量)，页码同步每3帧执行一次(原O(n)遍历)。提取`_sync_page_input()`独立方法。

**关键代码**: [DisplayWidget._rebuild_image_cache](guitar_tab_viewer.py), [DisplayWidget._hit_annotation/mousePressEvent/mouseMoveEvent/mouseReleaseEvent](guitar_tab_viewer.py), [_tick/_sync_page_input](guitar_tab_viewer.py)

### 3.5 循环播放模块

**原理**: 在 _tick() 回调中检测结束条件。

**三种模式**:
1. **不循环**: 到达末尾停止播放
2. **全局循环**: 到达末尾回到起点继续
3. **A-B区域循环**: 到达B点回到A点继续

**A-B设置方式**:
- Ctrl + 点击进度条 → 设置A点
- Shift + 点击进度条 → 设置B点

**关键代码**: [_check_loop_condition](guitar_tab_viewer.py), [_tick](guitar_tab_viewer.py)

## 4. 配置与存储

### 4.1 配置文件
- 路径: `config/settings.json`
- 内容: 上次打开的目录路径、窗口大小位置等

### 4.2 标注存储
- **新路径(默认)**: 与源文件同名 + `.anno.json`，存放在谱子同目录（方便分发/备份）
- **旧路径(兼容)**: `data/annotations/{hash}.json` (按文件内容哈希命名，仅读取不写入)
- 格式: JSON，Annotation 列表的字典序列化
- 加载优先级: 新路径 > 旧路径 > 空列表

### 4.3 路径约定
所有路径使用相对路径，基于 `os.path.dirname(os.path.abspath(__file__))` 计算，兼容 Windows/Linux/Docker。

## 5. 扩展指南

### 5.1 添加新文件格式支持
1. 在 `SUPPORTED_*_EXTENSIONS` 元组添加扩展名
2. 在 `LoadContentWorker.run()` 添加对应的解析逻辑
3. 返回 List[QPixmap] 或预览文本

### 5.2 添加新预设曲线
1. 在 `SpeedCurveEditor._apply_preset()` 字典中添加新条目
2. 定义 List[SpeedCurvePoint] 作为默认控制点

### 5.3 实现缩放功能
`_on_zoom_changed(val)` 已预留接口，可在此实现 display_widget 的缩放逻辑。
