# 功能更新记录

## v1.7.0 (2026-06-09) - 点击位置非线性时间映射修复

### Bug修复
- **点击谱面位置误差较大(提前)**: mousePressEvent用线性比例`pos/total*time`反推音频时间
  - 根因: `_build_playhead_timeline`建立的是**非线性映射**(密集区每像素对应少时间，稀疏区多时间)
  - 线性比例在密集区**高估时间**→ 音频跳到点击位置后面 → 用户感觉"提前了"
  - 修复: 新增`_scroll_pos_to_time()`方法，在timeline中对scroll_y做二分查找+插值(与_time_to_scroll_pos完全对称的逆运算)
  - GTP模式用精确反向查找，非GTP模式降级为线性比例

### 技术原理
- 正向映射: `time_ms → _time_to_scroll_pos() → 二分查找time → 插值scroll_y → 居中 → current_position`
- 反向映射: `current_position → _scroll_pos_to_time() → 还原raw_scroll_y → 二分查找scroll_y → 插值time_ms → seek`
- 两者完全对称，保证点击位置与播放位置精确一致

### 修改文件
- `guitar_tab_viewer.py` — 新增_scroll_pos_to_time(); mousePressEvent使用反向查找替代线性比例

## v1.6.9 (2026-06-09) - seek静音 + 点击位置修正 + ×按钮区域扩展

### Bug修复
- **播放中点击出现双重声音**: seek()停旧线程但未静音已发声音符，新旧音频叠加
  - 根因: seek()只设置_stop_flag+join线程，旧note_on事件已发到FluidSynth仍在发声
  - 修复: 新增`silence_all_notes()`方法(全通道noteoff+pitch_bend复位)，在seek()的stop_flag之前调用
- **点击位置误差较大(提前)**: mousePressEvent中错误地乘以比例系数`total_scroll_distance/total_h`
  - 根因: `current_position`和`total_scroll_distance`都是像素单位(paintEvent直接用-current_position做偏移)，不需要比例转换。该比例<1导致结果偏小
  - 修复: 去掉比例系数，改为 `new_position = click_abs_y - height/2`(点击处居中显示)
- **标注×删除按钮移到时消失**: ×按钮画在bg_rect右侧外部，鼠标移过去离开命中区域→悬停清除→按钮消失
  - 修复: `_hit_annotation()`右边界扩展28px(含×按钮宽度18px+间距+余量)

## v1.6.8 (2026-06-09) - synth_engine seek偏移修复 + 推弦pre-note-on复位

### Bug修复
- **点击谱面播放条回到开头(根因修复)**: synth_engine.current_time_ms属性计算公式不包含seek偏移
  - 根因: `current_time_ms = (perf_counter - _start_time) * 1000 - paused_duration`，完全忽略seek设置的_current_time_ms
  - 现象: seek到5000ms后play()，33ms后current_time_ms返回≈33ms而非5033ms → _time_to_scroll_pos→≈0
  - 修复: 新增`_initial_time_offset`实例变量，在play()/seek()/_play_loop()中同步维护
  - 公式改为: `current_time_ms = elapsed - paused + _initial_time_offset`
  - stop()中重置所有通道pitch_bend=8192 + 重置offset=0
- **推弦后后续音符音高变高(根本性修复)**: 改用"防御性复位"策略
  - 原方案(失败): note_off后发送pitch_bend=8192复位 — 有时序风险，可能被跳过或延迟
  - 新方案: **每个note_on前检查并复位pitch_bend** — 无论前一个音符的弯音状态如何，当前音符都以干净状态开始
  - 逻辑: 无推弦/推弦起点为0的音符 → note_on前发8192复位; 有推弦且从原位开始 → 不需要预先复位
  - 效果: 即使之前的复位事件被跳过(seek等)，下一个note_on也会自动修正音高

## v1.6.7 (2026-06-09) - 推弦渐变效果 + 音高泄漏修复

### Bug修复
- **推弦只有"咔"一声**: 原代码只发1个pitch_bend事件(8192→目标值)，MIDI弯音是瞬时跳变
  - 修复: 新增`_generate_bend_events()`方法，生成多个渐变pitch_bend事件
  - 策略: 使用BendData.points曲线点数据(如有)或自动生成3点渐变(0→半→峰)
  - 效果: 音高从原位平滑过渡到目标音高(模拟真实推弦手感)
- **推弦后后续音符全部变高**: pitch_bend复位时机/条件不正确
  - 修复: 无论是否有释放段，都在note_off后强制发送pitch_bend=8192复位事件
  - 复位时间: `beat_tick + actual_duration + beat_ticks//4`(确保在note_off之后)
  - 这是最终防线，防止任何情况下弯音状态泄漏

### 技术原理
- MIDI Pitch Bend是14位瞬时值(0-16383, 中值8192)，单次设置=突变
- 渐变效果需要多个中间值事件，间隔≥10ms(约30ticks@120BPM)
- 转换公式: 四分之一音×81.92 = MIDI偏移量 (Full=+8192, Half=+4096)

## v1.6.6 (2026-06-09) - 修复点击播放双重声音+执行顺序重构

### Bug修复
- **点击播放出现双重声音**: mousePressEvent中先调`_on_progress_changed`(内部seek)再调`toggle_playback`(内部play)导致两个播放线程
  - 根因: 执行顺序错误 + `_on_progress_changed`副作用(覆写current_position)
  - 修复: 重构为三步顺序: ①设位置+跳转目标 → ②直接seek(不用_on_progress_changed) → ③最后start_playback
  - 关键: seek()在play()之前调用，确保play()启动的线程使用正确的start_offset
- **点击后播放条回到开头**: _on_progress_changed覆写current_position + 音频时间从0开始覆盖
  - 修复: 不再使用_on_progress_changed，直接调用synth.seek()；保留_click_jump_target保护

### 修改文件
- `guitar_tab_viewer.py` — mousePressEvent重写为三步顺序(设位→seek→play)，直接调用synth_engine.seek()

## v1.6.5 (2026-06-09) - 修复推弦音高泄漏+点击跳转保护

### Bug修复
- **推弦后音高影响后续音符**: pitch_bend事件未在note_off时复位到8192(中值)
  - 现象: 推弦后的所有同通道音符音高都变高了
  - 根因: 只有`has_release=True`的推弦才发送复位事件，大部分推弦没有释放段
  - 修复: 无论是否有释放段，都在note_off后强制发送`pitch_bend=8192`复位事件
  - 避免重复: `has_release=True`时不额外发送(已由释放段处理)
- **点击谱面播放仍回到开头**: GTP模式下`_tick()`用音频时间覆盖点击位置
  - 现象: 点击谱面任意位置开始播放，但总是从开头开始
  - 根因: 点击设置current_position → seek音频 → start_playback() → _tick()读取audio_time_ms(≈0) → `_time_to_scroll_pos(≈0)`覆盖位置
  - 修复: 新增`_click_jump_target`标志位，_tick()在音频时间追上目标前保持使用点击位置
  - 策略: `time_based_pos < click_target`时保持点击位置；追上后清除标志恢复正常模式

## v1.6.3 (2026-06-09) - 多项Bug修复 + 推弦MIDI + 点击播放

### Bug修复
- **_playhead_timeline未初始化导致AttributeError**: DisplayWindow.__init__中缺少`self._playhead_timeline=[]`初始化
  - 现象: 打开图片/PDF文件时报错`'DisplayWindow' object has no attribute '_playhead_timeline'`
  - 修复: 在__init__中添加`self._playhead_timeline=[]`和`self._total_audio_duration_ms=0.0`初始化
  - 原因: _update_playhead()在_tick()中被调用，但_build_playhead_timeline()仅在GTP文件时执行
- **标注删除后意外新建**: 删除标注时使用对象引用(ann)而非ID匹配，可能导致引用失效
  - 修复: 改用`target_ann_id = ann.id`记录目标ID，删除/编辑时基于ID操作
  - 同时加强return保护，确保删除后不继续执行新建分支
- **标注命中区域不准确**: 使用粗略估算`len(text)*font_size//2+20`计算宽度
  - 修复: 改用QFontMetrics精确计算文字实际渲染宽度
  - 效果: 中英文混合、不同字体大小时点击区域精确匹配

### 新增功能
- **点击谱面跳转播放**: 单击谱面任意位置自动跳转到该位置并开始播放
  - 点击位置不在标注上时触发，计算scroll_y并同步音频seek
  - 如果尚未播放则自动开始播放
  - 实现原理: DisplayWidget.mousePressEvent新增点击跳转逻辑
- **推弦(Pitch Bend)MIDI支持**: 推弦技巧现在能正确转换为MIDI pitch_bend事件
  - MidiEvent新增`pitch_bend`类型，支持0-16383范围弯音值

  - SynthEngine._send_event()新增pitch_bend处理分支
  - 支持释放段: 推弦后回到原位(pitch_bend值回到8192)
- **GTP模式隐藏速度控制**: GTP文件模式下自动隐藏手动速度调节(ms/曲线)
  - 原因: GTP模式由BPM驱动播放，不需要手动速度控制

  - 技术原理: v1.6.0已改用固定33ms定时器+scroll_step控制速度，timer.start()会破坏固定帧率

- **悬停显示删除按钮(×)**: 鼠标悬停在标注上时，自动显示红色圆形×删除按钮

  - 点击×按钮可直接删除标注(支持撤销)

## v1.6.0 (2026-06-08) - 导出系统重构 + 播放平滑优化 + 时间驱动滚动

### Bug修复
- **导出PNG/PDF为0字节**: 使用QImage替代QPixmap作为画布缓冲区，确保`painter.end()`在`save()`之前调用，修复数据未flush导致空文件
- **播放条卡顿不连贯**: 根因是定时器间隔=base_speed(350-700ms)，每秒仅更新1-3次
  - 修复: 改用固定30fps定时器(33ms)，通过调整每帧scroll_step控制播放速度
  - `_calculate_scroll_step()`: 新公式使用固定FRAME_INTERVAL_MS=33替代base_speed
  - `start_playback()`: `timer.start(33)` 替代 `timer.start(base_speed)`
  - 效果: 播放条和谱面滚动从"跳一下动一下"变为平滑连续移动
- **播放条速度不随音符时值变化**: 根因是滚动位置线性恒速递增，与音乐时间脱节
  - 修复: 新增`_time_to_scroll_pos()`方法，用音频时间→时间线索引二分查找+线性插值获取滚动Y位置
  - `_build_playhead_timeline()`: 每个拍新增`scroll_y`字段(累计页面高度+系统Y偏移)
  - `_tick()`: 有音频引擎时切换为**时间驱动模式**(`synth_engine.current_time_ms` → scroll_pos)，无音频时降级为线性模式
  - 效果: 音符密集区(16/32分音符)自动加快滚动，稀疏区(全/二分音符)自动减慢，与实际节奏同步
- **ExportDialog类缩进错误**: 插入ExportDialog后后续方法被挤出DisplayWindow类体变成模块级函数
  - 修复: 将`_open_speed_curve_editor`等5个方法移回DisplayWindow类内部正确位置
- **QRadioButton未导入**: ExportDialog使用了QRadioButton但未在QtWidgets导入列表中

### 新增功能/重构
- **导出对话框(ExportDialog)**: 自定义对话框替代原生文件选择器，支持:
  - 格式选择: PNG(每页一张图片) / PDF(单文件多页)
  - 分轨导出(GTP多轨): 仅当前轨 / 全部轨道(按顺序拼接，文件名带tN后缀) / 指定轨道(checkbox多选)
  - 分页导出: 全部页面 / 指定范围(起始页~结束页)
- **分轨标注渲染**: 导出时自动加载对应轨道的独立标注文件(`.tN.anno.json`)
- **PNG每页一图**: 多页内容自动生成 `xxx_p1.png`, `xxx_p2.png`, ...
- **PDF单文件**: 所有页面合并在一个PDF中，自动分页
