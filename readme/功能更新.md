## (2026-06-19) - Linux 多音频驱动自动尝试 (解决无声音问题) 

### 修改内容

- **音频驱动自动尝试**: `initialize()` 方法在启动音频驱动时，按优先级依次尝试多种驱动
  - **Linux**: `pulseaudio` → `pipewire` → `alsa` → `jack` → `default`
  - **Windows**: `dsound` → `waveout` → `default`
  - **macOS**: `coreaudio` → `default`

- **失败提示优化**: 所有驱动均不可用时，打印详细的排查建议:
  - 未安装音频服务 (PulseAudio/PipeWire/ALSA)
  - Docker 容器内未挂载 /dev/snd 设备
  - SSH 远程连接时未配置音频转发

### 技术实现

- 使用 for 循环遍历驱动列表，每个驱动 try-except 包裹
- 首个成功启动的驱动即停止后续尝试
- 驱动启动状态实时打印到控制台方便调试

### 修改文件

| 文件 | 改动 |
| --- | --- |
| `ApolloTab/audio/synth_engine.py` | 重构音频驱动初始化逻辑; 新增多平台驱动列表; 更新文件头部注释 |

### 10条Case预期结果

| Case | 场景 | 预期结果 |

| 1 | Linux + PulseAudio 已安装 | 自动使用 pulseaudio 驱动成功播放 |
| 2 | Linux + PipeWire 已安装(无PulseAudio) | 自动降级到 pipewire 驱动成功播放 |

## (2026-06-19) - 跨平台 FluidSynth 库查找支持 (Windows DLL / Linux .so) 

### 修改内容

- **新增平台条件判断**: `initialize()` 方法根据操作系统自动选择库文件查找策略
  - **Windows**: 保持原有逻辑，查找 `libfluidsynth-3.dll` 并添加到 PATH / add_dll_directory
  - **Linux**: 新增 `_find_so_path()` 方法，按发行版自动搜索 `.so` 文件

- **Linux 多发行版支持** (`_find_so_path()` 方法):
  | 发行版 | 包管理器 | 搜索路径 |
  |--------|----------|----------|
  | Ubuntu/Debian x64 | apt | `/usr/lib/x86_64-linux-gnu/` |
  | Ubuntu/Debian ARM64 | apt | `/usr/lib/aarch64-linux-gnu/` |
  | Fedora/RHEL/CentOS | dnf/yum | `/usr/lib64/` |
  | Arch Linux/openSUSE/Alpine | pacman/zypper/apk | `/usr/lib/` |
  | 本地编译安装 | - | `/usr/local/lib/` |

- **搜索的 .so 文件名**（按版本优先级）:
  - `libfluidsynth.so.3` (FluidSynth v2.x/v3.x, 推荐)
  - `libfluidsynth.so.2` (FluidSynth v1.x/v2.x)
  - `libfluidsynth.so.1` (FluidSynth v1.x)
  - `libfluidsynth.so` (无版本号符号链接)

### 技术实现

- Linux 平台: 找到 .so 后设置 `LD_LIBRARY_PATH` 环境变量 + 使用 `ctypes.CDLL()` 预加载
- Windows 平台: 保持原有 PATH + add_dll_directory 方式不变
- 未找到时打印各发行版的安装命令提示

### 修改文件

| 文件 | 改动 |
| --- | --- |
| `ApolloTab/audio/synth_engine.py` | 新增 `platform` 导入; 重构 `initialize()` 添加平台分支; 新增 `_find_so_path()` 静态方法; 更新文件头部注释 |
| `TAB Score Viewer_linux.spec` | **新增** Linux 版 PyInstaller 打包配置文件 |
| `build_deb.sh` | **新增** DEB 包自动化构建脚本(含启动脚本+桌面快捷方式) |

## v2.0.7 (2026-06-15) - 异步导出+JPG格式支持

### 新功能

- **异步导出系统 (解决大文件UI冻结)**:
  - 导出操作改为后台线程(QRunnable + QThreadPool)执行，UI不再冻结
  - 新增 `ExportProgressDialog` 进度对话框:
    - 实时显示渲染状态(收集数据 / 渲染轨道X/Y / 保存文件 / 完成)
    - 进度条可视化展示整体进度
    - 取消按钮可随时中断导出
    - 完成/出错/取消后自动关闭并提示结果
  - 新增 `ExportWorker` 工作线程类: 通过信号报告进度/完成/错误/取消

- **JPG格式导出支持**:
  - ExportDialog格式选项新增 **JPG (有损压缩,适合分享)**
  - 新增JPG质量滑块(1-100，默认90)，仅选择JPG时显示
  - 质量参数说明: 90=高质量接近无损 / 80=网络分享 / 60=极致压缩
  - 文件体积比PNG小3-5倍，适合微信/QQ/论坛分享
  - 新增 `_render_to_a4_jpg()` 方法: 基于PNG渲染逻辑，使用Qt的JPEG质量参数输出

### 架构变更

- **导出流程从同步改为异步**:
  - 旧版(v2): `_export_to_a4()` → UI主线程直接调用渲染 → 大文件冻结5-10秒
  - 新版(v3): `_export_to_a4()` → 创建ExportWorker → QThreadPool后台运行 → 信号驱动进度更新
  - ExportWorker持有DisplayWindow引用，调用其渲染方法完成实际工作
  - ExportProgressDialog通过信号槽机制接收进度更新(跨线程安全)

### 技术实现

- **ExportWorkerSignals**: 自定义信号类(progress/finished/error/cancelled)，继承QObject支持pyqtSignal
- **ExportWorker(QRunnable)**: 
  - run()方法中依次执行: 收集数据 → 逐轨渲染(每轨前检查取消标志) → 报告完成
  - cancel()方法设置原子标志位，下个渲染循环生效
- **ExportProgressDialog(QDialog)**:
  - set_worker()绑定ExportWorker实例和信号连接
  - 进度条计算: 每轨道占(100/总轨道)分，当前轨道内按页码细分
  - 自动关闭: 完成500ms后 / 出错500ms后 / 取消800ms后
- **_render_to_a4_jpg()**: 
  - 复用PNG的预渲染防溢出方案(page_buffer → A4画布)
  - QImage格式用RGB32(无alpha通道，比ARGB32更省空间)
  - Qt质量映射: 用户1-100 → Qt内部0-99

### UI参数(可调整)

| 参数 | 默认值 | 说明 | 调整效果 |
|------|--------|------|----------|
| jpg_quality | 90 | JPG压缩质量(1-100) | 越大画质越好文件越大 |
| progress_dialog_width | 420px | 进度对话框宽度 | 影响对话框大小 |
| progress_dialog_height | 160px | 进度对话框高度 | 影响对话框大小 |
