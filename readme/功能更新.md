# 功能更新记录

## v2.0.1 (2026-06-13) - Bug修复 + UI优化

### UI优化
- **移除缩放功能**: 移除工具栏中的缩放滑块(预留接口`_on_zoom_changed`已移除)，简化界面
  - 原因: 缩放功能未实际实现(方法体为pass)，UI上仅显示无作用的滑块
- **移除所有Emoji图标**: 按照专业UI设计规范(ui-ux-pro-max)，移除按钮/菜单中的emoji字符
  - 影响范围: zh_CN.json / en_US.json 全部翻译文本 + 主程序注释
  - 替换方案: 使用纯文字标签（如"标注"替代"📝 标注"，"导出"替代"💾 导出"）
  - 设计原则: 专业UI应使用SVG图标或纯文字，不使用emoji作为UI图标

### 打包优化
- **SoundFont音色库打包支持**: spec文件新增soundfont/目录到datas列表
  - 打包后exe所在目录自动包含soundfont/FluidR3_GM.sf2
- **音色库路径自适应**: `_init_audio_engine()` 新增搜索路径注入逻辑
  - 开发模式: 自动添加 `脚本目录/soundfont/` 到ApolloTab搜索列表
  - 打包模式: 自动添加 `exe所在目录/soundfont/` 到ApolloTab搜索列表
  - 兼容性: 无论从哪个CWD启动程序，都能正确找到音色库文件

### Bug修复
- **GTP模式点击谱面前几行无法跳转进度**:
  - v2.0.1a问题: `total_scroll_distance=0`时整个点击处理被跳过
  - v2.0.1b问题: `height/2`居中偏移→点击前几行截断为0
  - v2.0.1c问题: `height*0.25`偏移→position大时点击上部仍会后移("跳到很后面")
  - 根因: `click_abs_y = position + y`, 任何非零偏移都会导致意外跳动
  - 迭代历史(4次):
    1. height/2居中 → 点前几行截断为0(无法跳转)
    2. height*0.25 → position大时点上部仍会后移("跳到很后面")
    3. 固定80px → 前几行跳2~3行
    4. 动态max(10,y*0.14) → 更严重
  - 最终修复: **完全去掉偏移(offset=0)**, new_position=click_abs_y, 精确定位到点击处
    + toggle_playback提到条件外层(确保短谱也能启动播放)

### 技术原理
- **零偏移策略**: `offset=0`, `new_position = click_abs_y = position + event.y()`, 精确映射
- **播放启动解耦**: toggle_playback从条件内提到外层，任何情况下点击都能启动/继续播放
- **行为**: 点击哪行→该行跑到屏幕最顶部→seek到对应时间→开始播放,无任何额外跳动

| Case | 场景 | 预期结果 |
|------|------|----------|

## v2.0.0 (2026-06-13) - 深色/浅色主题系统 + UI美化

### 新功能
- **ThemeManager 主题管理器**: 全局深色/浅色主题统一管理
  - **单例+QObject**: 继承QObject，`pyqtSignal(str)` 作为类属性定义（符合PyQt5规范）
  - **动态切换**: 运行时实时切换深色/浅色主题，无需重启应用
  - **信号通知**: `theme_changed` 信号发射，DisplayWindow自动连接并刷新全部页面
  - **GTP渲染联动**: UI主题与ApolloTab渲染主题自动同步
    - 浅色UI → GTP谱使用 light 渲染主题
    - 深色UI → GTP谱使用 dark 渲染主题
    - 导出GTP谱 → 强制使用 light 浅色渲染（适合打印）

- **浅色主题配色方案(THEME_LIGHT)**: 精心设计的浅色模式配色
- **自定义组件主题适配**: ModernButton/ModernSlider/ProgressBarSlider 全部从 ThemeManager.get() 获取颜色
- **主题持久化**: settings.json 的 `theme` 字段，启动自动恢复
- **国际化支持**: theme_label / theme_tooltip 中英文翻译

### Bug修复
- **GTP模式暂停恢复播放**: 暂停后再播放从暂停位置继续，而非回到开始
  - 新增 `_paused_time_ms` 变量保存暂停时的播放时间
  - `stop_playback()` 新增 `reset_position` 参数区分暂停/停止
  - 播放按钮(暂停): 保存当前位置，恢复时继续
  - 停止按钮: 重置到开始位置
- **LoadContentWorker 方法缺失**: 新增 `_create_error_image()` 方法，修复加载失败时的错误显示
- **GTPPlayer.set_theme 兼容性**: 添加 `hasattr()` 检查，兼容不支持主题切换的旧版 GTPPlayer
- **ThemeManager pyqtSignal 规范修复**:
  - Issue1: 未继承 QObject → 改为 `class ThemeManager(QObject)` + `super().__init__()`
  - Issue2: 动态赋值实例属性 → `theme_changed = pyqtSignal(str)` 提升为类属性
  - 删除 `init_signal()` 延迟初始化方法（不再需要）
- **导出跨页溢出修复(v4)**:
  - PNG/PDF导出使用 `QPainter.setClipRect()` 绝对裁剪，彻底消除页底溢出
  - 边距从2%缩小到0.5%，图片间距从5px减到3px，最大化谱面显示区域
  - 预留30px页码区域，不与内容重叠
- **GTPPlayer.is_loaded 属性缺失**: ApolloTab/player.py 新增 `@property is_loaded`
- **导出强制浅色主题**: 当前轨(track_mode="current")导出路径也新建renderer+set_theme('light')
- **工具栏GTP主题按钮已移除**: GTP渲染主题完全跟随UI主题自动同步

### 技术原理
- **设计模式**: 单例模式(Singleton) + 观察者模式(Observer/pyqtSignal)
- **PyQt5信号规范**: pyqtSignal 必须是类属性 + 必须在QObject子类中定义
- **绝对裁剪**: setClipRect() 在渲染管线层面设置硬边界，不受浮点舍入影响

### 修改文件
- **重构** `TAB Score Viewer.py` — ThemeManager(QObject) + THEEM_LIGHT + _refresh_theme() + 导出v4防溢出
- **修改** `ApolloTab/player.py` — 新增 @property is_loaded + set_theme() 代理方法
- **更新** `locales/zh_CN.json`, `locales/en_US.json` — theme相关翻译
