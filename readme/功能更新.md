## (2026-06-20) - 最近打开文件功能 (独立列表控件)

### 修改内容

- **最近文件列表(独立控件)**: 与文件夹文件列表完全分离的独立 QListWidget，支持折叠/展开
  - 位置: 文件夹文件列表上方，带标题行(标签+展开/折叠按钮)，两列表之间间距 8px
  - **默认折叠**: 只显示1行内容，点击"展开 ▼"按钮显示全部
  - 显示格式: `📄 文件名` (使用强调色/橙色突出显示)
  - 边框样式: 使用强调色边框(与普通文件列表的主色边框区分)
  - 高度自适应: 折叠时1行高度，展开时紧贴所有项数
  - 单击即可快速重新打开(比普通双击更便捷)
  - 鼠标悬停显示完整路径提示
  - 右键菜单: 打开文件 / 在资源管理器中定位 / 从列表移除

- **数据持久化**: 
  - 存储位置: `config/settings.json` 的 `recent_files` 字段
  - 存储内容: 绝对路径列表(最多4条)
  - 自动过滤不存在的文件(启动时清理)

- **自动更新逻辑**:
  - 每次通过 `show_display()` 打开文件时自动记录
  - 新文件插入列表头部(最新的在最前面)
  - 自动去重: 已存在的文件先移除再插入头部
  - 自动截断: 超过4条时删除最旧的记录
  - 仅记录单个文件路径(PDF/GTP/单张图片)，不记录图片列表

### 技术实现

- `_add_recent_file()`: 添加文件到最近列表(去重+截断+持久化+刷新UI)
- `_refresh_recent_list()`: 刷新独立的 recent_list 控件内容
- `_apply_recent_list_height()`: 根据折叠/展开状态设置精确高度
- `_toggle_recent_list()`: 切换展开/折叠状态(翻转布尔值+更新按钮文字+应用高度)
- `_on_recent_file_clicked()`: 最近文件单击事件处理(仅响应recent_list)
- `_show_recent_context_menu()`: 最近文件右键菜单(打开/定位/移除)
- `_remove_recent_file()`: 从最近列表移除指定文件
- `load_config()/save_config()`: 扩展支持 recent_files 字段
- CSS选择器区分: `#recent_list_widget`(强调色) vs `#file_list_widget`(主色)

### UI参数(可调整)

| 参数 | 默认值 | 位置 | 说明 | 调整效果 |
|------|--------|------|------|----------|
| MAX_RECENT_FILES | 4 | SettingsWindow类属性 | 最大保存数量 | 越大占用更多配置空间 |
| _recent_expanded | False | init_ui() | 默认展开/折叠 | True=默认展开，False=默认折叠 |
| 列表间距 | 8px | init_ui() addSpacing | 两列表之间间距 | 可调范围5-10px |
| 按钮高度 | 24px | recent_toggle_btn | 展开/折叠按钮高度 | 增大更易点击 |

## (2026-06-19) - Linux 多音频驱动自动尝试 (解决无声音问题) 

### 修改内容

- **音频驱动自动尝试**: `initialize()` 方法在启动音频驱动时，按优先级依次尝试多种驱动
  - **Linux**: `pulseaudio` → `pipewire` → `alsa` → `jack` → `default`
  - **Windows**: `dsound` → `waveout` → `default`
  - **macOS**: `coreaudio` → `default`

- **失败提示优化**: 所有驱动均不可用时，打印详细的排查建议:
  - 未安装音频服务 (PulseAudio/PipeWire/ALSA)
  - Docker 容器内未挂载 /dev/snd 设备
  - SSH 远程连接时未配置音频转发

### 技术实现

- 使用 for 循环遍历驱动列表，每个驱动 try-except 包裹
- 首个成功启动的驱动即停止后续尝试
- 驱动启动状态实时打印到控制台方便调试

### 修改文件

| 文件 | 改动 |
| --- | --- |
| `ApolloTab/audio/synth_engine.py` | 重构音频驱动初始化逻辑; 新增多平台驱动列表; 更新文件头部注释 |

## (2026-06-19) - 跨平台 FluidSynth 库查找支持 (Windows DLL / Linux .so) 

### 修改内容

- **新增平台条件判断**: `initialize()` 方法根据操作系统自动选择库文件查找策略
  - **Windows**: 保持原有逻辑，查找 `libfluidsynth-3.dll` 并添加到 PATH / add_dll_directory
  - **Linux**: 新增 `_find_so_path()` 方法，按发行版自动搜索 `.so` 文件

- **Linux 多发行版支持** (`_find_so_path()` 方法):
  | 发行版 | 包管理器 | 搜索路径 |
  |--------|----------|----------|
  | Ubuntu/Debian x64 | apt | `/usr/lib/x86_64-linux-gnu/` |
  | Ubuntu/Debian ARM64 | apt | `/usr/lib/aarch64-linux-gnu/` |
  | Fedora/RHEL/CentOS | dnf/yum | `/usr/lib64/` |
  | Arch Linux/openSUSE/Alpine | pacman/zypper/apk | `/usr/lib/` |
  | 本地编译安装 | - | `/usr/local/lib/` |

- **搜索的 .so 文件名**（按版本优先级）:
  - `libfluidsynth.so.3` (FluidSynth v2.x/v3.x, 推荐)
  - `libfluidsynth.so.2` (FluidSynth v1.x/v2.x)
  - `libfluidsynth.so.1` (FluidSynth v1.x)
  - `libfluidsynth.so` (无版本号符号链接)

### 技术实现

- Linux 平台: 找到 .so 后设置 `LD_LIBRARY_PATH` 环境变量 + 使用 `ctypes.CDLL()` 预加载
- Windows 平台: 保持原有 PATH + add_dll_directory 方式不变
- 未找到时打印各发行版的安装命令提示

### 修改文件

| 文件 | 改动 |
| --- | --- |
| `ApolloTab/audio/synth_engine.py` | 新增 `platform` 导入; 重构 `initialize()` 添加平台分支; 新增 `_find_so_path()` 静态方法; 更新文件头部注释 |
| `TAB Score Viewer_linux.spec` | **新增** Linux 版 PyInstaller 打包配置文件 |
| `build_deb.sh` | **新增** DEB 包自动化构建脚本(含启动脚本+桌面快捷方式) |

- **异步导出系统 (解决大文件UI冻结)**:
  - 导出操作改为后台线程(QRunnable + QThreadPool)执行，UI不再冻结
  - 新增 `ExportProgressDialog` 进度对话框: 实时显示渲染状态/进度条/取消按钮
