## v2.0.5 (2026-06-14) - 打印曲谱功能(含标注，支持分轨/分页)

### 新功能

- **打印曲谱到打印机**:
  - 工具栏新增"打印"按钮(打印机图标)，点击后弹出打印设置对话框
  - 支持选择轨道(GTP多轨时: 当前轨/全部轨/指定轨道)
  - 支持选择页码范围(全部页面/指定范围)
  - 弹出系统打印对话框(QPrintDialog)让用户选择打印机
  - 自动分页: 内容超过一页A4时自动分多页打印
  - **包含所有标注**: 完整渲染标注(颜色/字体/大小/背景色)，排序与导出功能完全一致

### 技术实现

- **复用导出逻辑**: 打印渲染核心`_render_to_printer()`完全复用A4导出的缩放+预渲染防溢出方案
  - 按打印机可打印区域宽度等比缩放图片
  - 使用"预渲染+合成"方案(page_buffer → QPrinter)杜绝内容溢出
  - 标注绘制复用`_draw_annotations_on_canvas()`方法
  - 跳过近空白尾页检测(阈值5%)
- **对话框**: 新增`PrintDialog`类(基于ExportDialog简化，移除格式选择部分)
- **图标**: 新增`icons/printer.svg`(Lucide风格打印机图标)

## v2.0.4 (2026-06-14) - A/B循环架构重构(下沉到库层，基于小节原子单位)

### 架构变更

- **A/B循环逻辑从UI层完全移除，下沉到库层(SynthEngine音频线程内部)**:
  - 用户核心思路: "把每个小节都作为一个模块，A在哪个小节就从那个小节开始播，B在哪个小节就在那个小节末尾结束"
  - **旧方案的问题**(v2.0.3及之前): UI层用帧计数器/冷却/模拟时钟/安全边界等复杂机制试图解决竞态→每修一个bug引出新的→无限循环
  - **新方案的原理**:
    1. `SynthEngine._play_loop()` 内置循环: 播放完所有事件后检查`_loop_enabled`标志 → 若启用则静音→重置时间基准→重新遍历事件列表（早于loop\_start的事件被time检查自动跳过）
    2. `GTPPlayer` 提供基于小节的API: `set_loop_region_by_position(pct, pct)` 将百分比→转换为小节索引→对齐到小节边界(起始小节第一拍/结束小节最后一拍)
    3. UI层 `_tick()` 恢复为干净的"读audio\_time\_ms→算position"，零循环相关代码
  - **彻底消除的问题**: 竞态条件、冷却期、模拟时钟、安全边界、停顿、跳动、卡死 — 全部不存在了

### 修改文件

| 文件                                | 改动                                                                                                                                                                                                              |
| --------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `ApolloTab/audio/synth_engine.py` | 新增 `_loop_enabled/_loop_start_ms/_loop_end_ms` 属性; 新增 `set_loop_region()/clear_loop_region()` 方法; `_play_loop()` 从for改为while循环，支持内部循环重启                                                                         |
| `ApolloTab/player.py`             | 新增 `set_loop_region()`, `set_loop_region_by_measure()`, `set_loop_region_by_position()`, `clear_loop_region()`, `_find_measure_at_time()`, `is_loop_enabled` 属性                                                 |
| 主文件                               | **删除**: `_loop_seek_cooldown/_target_ms/_sim_time` 变量, `_execute_loop_seek()` 方法; **简化** `_tick()` 时间驱动段(\~110行复杂逻辑→\~25行干净代码); **重写** `_on_loop_mode_changed()`/`_set_ab_point()`/`_clear_ab_points()` 使用库层API |

### 10条Case预期结果

| Case | 场景                     | 预期结果                  |
| ---- | ---------------------- | --------------------- |
| 1    | 第1小节短循环(A-B跨1-2个小节)    | 平滑A→B→A循环，无卡顿无跳动      |

| 6    | 播放中切换到A-B模式（当前位置在AB之间） | 继续播放，到达B后由库层自动循环回A    |

### 增量修复 (v2.0.4 迭代调试中修复的6个问题)

| # | 问题                                      | 根因                                                     | 修复                                                  |
| - | --------------------------------------- | ------------------------------------------------------ | --------------------------------------------------- |
| 1 | **设A/B按钮pct全是0%** → 小节\[0-0]            | `_set_ab_point()`用UI滚动位置`current_position`(未播放时=0)算百分比 | 改为优先用GTP音频时间 `audio_time_ms / total_duration_ms`    |
| 2 | **切换到region模式后不跳回A点**                   | `set_loop_region()`只设参数不seek，音频继续在当前位置播                | `_on_loop_mode_changed()`增加立即`seek(target_ms)`到A点   |
| 3 | **短循环先播完整首谱才回来**                        | 库层循环只在所有事件播完后才触发(while结束条件)                            | 新增事件级检查: `target_time_ms >= loop_end_ms` 时立即静音+重启循环 |
| 4 | **B点设在谱末尾附近返回meas\_idx=0**              | 时间线哨兵点(末尾虚拟点)的meas\_idx硬编码为0                           | 哨兵点继承最后1个真实条目的索引值                                   |
| 5 | **进度条Ctrl/Shift设AB后不同步库层**              | `_on_region_selected()`只更新了UI没调库层API                   | region模式下立即同步调用`set_loop_region_by_position()`      |
| 6 | **seek()竞态: 返回后读current\_time\_ms异常偏大** | 音频线程还没来得及设置`_start_time`                               | `seek()`Step4.5在锁内预置`_start_time`                   |
