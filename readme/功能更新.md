# 功能更新记录

## v1.6.5 (2026-06-09) - 修复推弦音高泄漏+点击跳转保护

### Bug修复
- **推弦后音高影响后续音符**: pitch_bend事件未在note_off时复位到8192(中值)
  - 现象: 推弦后的所有同通道音符音高都变高了
  - 根因: 只有`has_release=True`的推弦才发送复位事件，大部分推弦没有释放段
  - 修复: 无论是否有释放段，都在note_off后强制发送`pitch_bend=8192`复位事件
  - 避免重复: `has_release=True`时不额外发送(已由释放段处理)
- **点击谱面播放仍回到开头**: GTP模式下`_tick()`用音频时间覆盖点击位置
  - 现象: 点击谱面任意位置开始播放，但总是从开头开始
  - 根因: 点击设置current_position → seek音频 → start_playback() → _tick()读取audio_time_ms(≈0) → `_time_to_scroll_pos(≈0)`覆盖位置
  - 修复: 新增`_click_jump_target`标志位，_tick()在音频时间追上目标前保持使用点击位置
  - 策略: `time_based_pos < click_target`时保持点击位置；追上后清除标志恢复正常模式

### 修改文件
- `gtp_engine/audio/midi_converter.py` — note_off后无条件发送pitch_bend复位(无has_release时)
- `guitar_tab_viewer.py` — 新增_click_jump_target变量; mousePressEvent设置跳转目标; _tick()增加跳转保护逻辑; stop_playback()清除标志

## v1.6.4 (2026-06-09) - 修复点击播放+分离标注窗口+命中区域修正

### Bug修复
- **点击谱面播放从头开始**: 原计算`current_position + click_y - height/2`逻辑错误
  - 现象: 点击谱面任意位置都从开头开始播放
  - 修复: 改为基于鼠标Y在总内容高度中的相对比例 → 映射到scroll_y范围
  - 公式: `click_abs_y = current_position + event.y()` → `new_position = click_abs_y * total_scroll_distance / total_h`
- **创建/编辑标注共用同一对话框导致删除变新建**: 根因是AnnotationEditDialog同时用于两种场景
  - 修复: 拆分为两个独立对话框:
    - `AnnotationCreateDialog`: 新建标注专用(标题"新建标注"，按钮[✓ 创建标注][取消])
    - `AnnotationEditDialog`: 编辑标注专用(标题"编辑标注"，按钮[确定保存][🗑 删除此标注][取消])
- **标注命中区域不准确**: 使用粗略估算而非实际描边范围
  - 修复: `_hit_annotation()`和`mouseDoubleClickEvent()`统一使用与`_draw_one_ann`完全相同的bg_rect计算
  - 命中区域 = `QRect(x-pad, y-th-pad, tw+2*pad, th+2*pad)` + 圆点高度6px

### 修改文件
- `guitar_tab_viewer.py` — mousePressEvent重写位置计算; 新增AnnotationCreateDialog类; AnnotationEditDialog改为编辑专用; _hit_annotation改用bg_rect; mouseDoubleClickEvent使用CreateDialog+bg_rect检测

## v1.6.3 (2026-06-09) - 多项Bug修复 + 推弦MIDI + 点击播放

### Bug修复
- **_playhead_timeline未初始化导致AttributeError**: DisplayWindow.__init__中缺少`self._playhead_timeline=[]`初始化
  - 现象: 打开图片/PDF文件时报错`'DisplayWindow' object has no attribute '_playhead_timeline'`
  - 修复: 在__init__中添加`self._playhead_timeline=[]`和`self._total_audio_duration_ms=0.0`初始化
  - 原因: _update_playhead()在_tick()中被调用，但_build_playhead_timeline()仅在GTP文件时执行
- **标注删除后意外新建**: 删除标注时使用对象引用(ann)而非ID匹配，可能导致引用失效
  - 修复: 改用`target_ann_id = ann.id`记录目标ID，删除/编辑时基于ID操作
  - 同时加强return保护，确保删除后不继续执行新建分支
- **标注命中区域不准确**: 使用粗略估算`len(text)*font_size//2+20`计算宽度
  - 修复: 改用QFontMetrics精确计算文字实际渲染宽度
  - 效果: 中英文混合、不同字体大小时点击区域精确匹配

### 新增功能
- **点击谱面跳转播放**: 单击谱面任意位置自动跳转到该位置并开始播放
  - 点击位置不在标注上时触发，计算scroll_y并同步音频seek
  - 如果尚未播放则自动开始播放
  - 实现原理: DisplayWidget.mousePressEvent新增点击跳转逻辑
- **推弦(Pitch Bend)MIDI支持**: 推弦技巧现在能正确转换为MIDI pitch_bend事件
  - MidiEvent新增`pitch_bend`类型，支持0-16383范围弯音值
  - BendData→MIDI转换: 1/4音=+2048, 1/2音=+4096, Full=+8192(中值8192)
  - SynthEngine._send_event()新增pitch_bend处理分支
  - 支持释放段: 推弦后回到原位(pitch_bend值回到8192)
- **GTP模式隐藏速度控制**: GTP文件模式下自动隐藏手动速度调节(ms/曲线)
  - 原因: GTP模式由BPM驱动播放，不需要手动速度控制
  - 隐藏组件: speed_spin, curve_btn, speed_group_box, curve_status_label
  - 图片/PDF模式仍显示这些控件

### 修改文件
- `guitar_tab_viewer.py` — __init__初始化_playhead_timeline; mouseDoubleClickEvent改用ID匹配; _hit_annotation使用QFontMetrics; mousePressEvent添加点击播放; _on_content_loaded控制速度控件可见性
- `gtp_engine/audio/midi_converter.py` — MidiEvent支持pitch_bend类型; _convert_measure生成推弦事件; 新增_bend_to_midi_pitch()转换方法
- `gtp_engine/audio/synth_engine.py` — _send_event()新增pitch_bend处理分支

## v1.6.2 (2026-06-09) - 修复速度调整卡顿 + 标注删除功能

### Bug修复
- **调整播放速度导致播放条卡顿**: 根因是`_on_speed_changed()`在播放时调用`timer.start(val)`，将33ms定时器重置为350-700ms
  - 现象: 拖动速度滑块后播放条明显卡顿(帧率从30fps骤降到1-3fps)
  - 修复: 移除`timer.start(val)`调用，`base_speed`变化通过`_tick()`中的`time_scale`自动生效
  - 效果: 调整速度时播放条保持30fps平滑滚动，速度变化立即生效无卡顿
  - 技术原理: v1.6.0已改用固定33ms定时器+scroll_step控制速度，timer.start()会破坏固定帧率

### 新增功能
- **编辑标注对话框添加删除按钮**: AnnotationEditDialog新增"🗑 删除此标注"按钮(红色)
  - 点击后弹出确认对话框，确认后删除标注并支持Ctrl+Z撤销
  - 对话框底部按钮布局: [确定保存] [🗑 删除此标注] [取消]
- **悬停显示删除按钮(×)**: 鼠标悬停在标注上时，自动显示红色圆形×删除按钮
  - 按钮位置: 标注文字右侧，垂直居中对齐
  - 点击×按钮可直接删除标注(支持撤销)
  - 鼠标移出标注区域时自动隐藏按钮
  - 实现原理: DisplayWidget新增`_hover_ann_id`悬停状态跟踪 + `_delete_btn_rect`点击区域记录

### 修改文件
- `guitar_tab_viewer.py` — _on_speed_changed()移除timer.start(); AnnotationEditDialog新增should_delete标记+删除按钮+确认弹窗; DisplayWidget新增_hover_ann_id/_delete_btn_rect状态变量; mouseMoveEvent增加悬停检测; mouseReleaseEvent增加删除按钮点击处理; _draw_one_ann增加is_hover参数和×按钮绘制; mouseDoubleClickEvent支持从编辑对话框删除

## v1.6.1 (2026-06-09) - 修复全轨并轨模式切轨播放条不同步

### Bug修复
- **全轨并轨模式切换轨道时播放条速度不同步**: 根因是切轨时仅更新了图像和布局数据，但未重建播放光标时间线(`_playhead_timeline`)
  - 现象: 切换轨道后滚动位置与当前显示的谱子不匹配，视觉播放速度错误
  - 修复: 在`_on_gtp_track_changed()`的并轨模式分支中添加`_build_playhead_timeline()`调用
  - 效果: 切轨后时间线立即基于新轨道的布局数据重建，滚动位置与谱子、MIDI音频完全同步
  - 技术原理: `_build_playhead_timeline()`遍历`_page_layouts`(新轨道)重新计算每个拍的time_ms→scroll_y映射，后续`_tick()`中的`_time_to_scroll_pos()`使用正确的映射关系

### 修改文件
- `guitar_tab_viewer.py` — _on_gtp_track_changed()并轨模式分支新增_build_playhead_timeline()调用(第1954行)

## v1.6.0 (2026-06-08) - 导出系统重构 + 播放平滑优化 + 时间驱动滚动

### Bug修复
- **导出PNG/PDF为0字节**: 使用QImage替代QPixmap作为画布缓冲区，确保`painter.end()`在`save()`之前调用，修复数据未flush导致空文件
- **播放条卡顿不连贯**: 根因是定时器间隔=base_speed(350-700ms)，每秒仅更新1-3次
  - 修复: 改用固定30fps定时器(33ms)，通过调整每帧scroll_step控制播放速度
  - `_calculate_scroll_step()`: 新公式使用固定FRAME_INTERVAL_MS=33替代base_speed
  - `start_playback()`: `timer.start(33)` 替代 `timer.start(base_speed)`
  - 效果: 播放条和谱面滚动从"跳一下动一下"变为平滑连续移动
- **播放条速度不随音符时值变化**: 根因是滚动位置线性恒速递增，与音乐时间脱节
  - 修复: 新增`_time_to_scroll_pos()`方法，用音频时间→时间线索引二分查找+线性插值获取滚动Y位置
  - `_build_playhead_timeline()`: 每个拍新增`scroll_y`字段(累计页面高度+系统Y偏移)
  - `_tick()`: 有音频引擎时切换为**时间驱动模式**(`synth_engine.current_time_ms` → scroll_pos)，无音频时降级为线性模式
  - 效果: 音符密集区(16/32分音符)自动加快滚动，稀疏区(全/二分音符)自动减慢，与实际节奏同步
- **ExportDialog类缩进错误**: 插入ExportDialog后后续方法被挤出DisplayWindow类体变成模块级函数
  - 修复: 将`_open_speed_curve_editor`等5个方法移回DisplayWindow类内部正确位置
- **QRadioButton未导入**: ExportDialog使用了QRadioButton但未在QtWidgets导入列表中

### 新增功能/重构
- **导出对话框(ExportDialog)**: 自定义对话框替代原生文件选择器，支持:
  - 格式选择: PNG(每页一张图片) / PDF(单文件多页)
  - 分轨导出(GTP多轨): 仅当前轨 / 全部轨道(按顺序拼接，文件名带tN后缀) / 指定轨道(checkbox多选)
  - 分页导出: 全部页面 / 指定范围(起始页~结束页)
- **分轨标注渲染**: 导出时自动加载对应轨道的独立标注文件(`.tN.anno.json`)
- **PNG每页一图**: 多页内容自动生成 `xxx_p1.png`, `xxx_p2.png`, ...
- **PDF单文件**: 所有页面合并在一个PDF中，自动分页

### 修改文件
- `guitar_tab_viewer.py` — 新增 ExportDialog 类; 重写 _export_to_a4 / _collect_export_data / _get_page_range_images / _load_track_annotations; 新增 _render_to_a4_png_v2(QImage+painter.end修复) / _render_to_a4_pdf_v2(setFileName修复); start_playback/timer.start改为33ms固定帧率; _calculate_scroll_step公式重写; 新增 _time_to_scroll_pos(时间→滚动映射) + _build_playhead_timeline(scroll_y字段); _tick()双模式(时间驱动/线性降级); 导入增加QRadioButton

## v1.5.0 (2026-06-07) - 标注拖动 + 速度范围调整 + 播放性能优化

### 新增功能
- **标注拖动移动**: 左键按住标注可拖动到谱面任意位置
  - 悬停标注时显示手型光标(OpenHandCursor)，拖动中显示抓手(ClosedHandCursor)

### 调整
- **速度范围调整为350-700ms**: 原范围1-500ms太宽且默认值过小导致播放过快
  - 新默认值: 500ms(中等速度)
  - 键盘调速步进: ±25ms(原±5ms，适配新范围)
  - SettingsWindow速度滑块映射同步更新到350-700区间

### 性能优化
- **图片缩放缓存**: DisplayWidget新增`_cached_scaled`缓存列表
  - paintEvent不再每帧调用`scaled()`(CPU密集操作)，改用预缓存图片
  - 窗口大小变化(resizeEvent)或图片更换(set_images)时自动重建缓存
- **UI更新节流**: `_tick()`中的进度显示优化
  - 进度条位置/百分比文字: 每帧更新(轻量操作，保持流畅)
  - 页码同步(`_sync_page_input`): 每3帧执行一次(原每帧O(n)遍历所有图片计算高度)
  - 提取`_sync_page_input()`独立方法，供_tick节流和update_progress_display全量更新共用

### 修改文件
- `guitar_tab_viewer.py` — DisplayWidget新增mousePressEvent/mouseMoveEvent/mouseReleaseEvent/_hit_annotation(拖动); _cached_scaled/_rebuild_image_cache/resizeEvent(图片缓存); _tick重构(内联轻量UI+_sync_page_input节流); speed_spin.setRange(350,700); show_display速度映射更新; 默认base_speed=500

## v1.4.0 (2026-06-07) - 标注导入导出 + 全局撤销重做 + A4导出 + 导航修复

### 新增功能
- **标注自动导入(同名文件)**: 打开谱子时自动加载同目录下同名 `.anno.json` 标注文件
  - 命名规则: `晚安北京.gp4` → `晚安北京.gp4.anno.json`（与谱子放一起，方便分发/备份）
  - 兼容旧路径: 优先加载新路径，找不到则回退到 `data/annotations/` 目录
- **标注实时保存**: 每次添加/修改/删除标注后自动写入 `.anno.json` 文件
- **A4导出(PNG+PDF，含标注渲染)**: 工具栏新增"导出"按钮，支持:
  - PNG格式: 高清位图(2480×3508px @300dpi)，适合查看/打印/分享
  - PDF格式: 矢量质量最佳，适合正式文档
  - 自动分页: 内容超过一页A4高度时自动分多页导出
  - 完整包含: 谱面图像 + 全部标注文字(颜色/字体/大小均保留)
- **全局统一撤销重做(Ctrl+Z/Y)**:
  - DisplayWindow内置全局撤销栈(`_undo_stack`)和重做栈(`_redo_stack`)
  - 画布双击添加标注、管理器内增删改操作、全部共享同一个撤销栈
  - 最大深度50步，超出自动丢弃最早记录
  - 快捷键: `Ctrl+Z` 撤销 / `Ctrl+Y` 重做（在DisplayWindow全局生效）

### Bug修复
- **资源管理器导航到文档文件夹**: `open_file_location()` 现在强制使用绝对路径(`os.path.abspath()`)，
  并改用`shell=True`调用explorer命令。相对路径会导致Windows资源管理器打开默认的"文档"文件夹而非目标位置。

### 技术原理
- 标注文件命名策略: 使用与源文件同名+`.anno.json`后缀，存放在谱子同目录
- 全局撤销栈设计: Command Pattern快照模式 — 每次变更前深拷贝当前annotations列表存入undo_stack
- A4导出流程: QPainter → QPixmap(QPNG) / QPdfWriter(PDF) → 绘制图片层 → 映射坐标绘制标注层 → 页码
- 导出坐标映射: 标注相对坐标(0-1) × 总内容高度 = 导出画布绝对像素位置

### 修改文件
- `guitar_tab_viewer.py` — 新增 `_export_to_a4()`/`_render_to_a4_png()`/`_render_to_a4_pdf()`/`_draw_annotations_on_canvas()`/`_get_a4_size_px()`; 重构 `_get_annotation_file_path()`/`_load_annotations()`/`_save_annotations()` 支持新旧双路径; 新增 `_anno_save_snapshot()`/`_anno_undo()`/`_anno_redo()` 全局撤销栈; AnnotationManagerDialog委托撤销到父窗口; keyPressEvent增加Ctrl+Z/Y; open_file_location修复绝对路径
