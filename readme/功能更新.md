# 功能更新记录

## (2026-06-20) - 反复记号(Repeat Signs)播放支持 (含嵌套反复)

### 修改内容

- **反复记号展开引擎**: 新增基于栈结构的嵌套反复记号展开算法
  - 支持简单反复: `||: A B :||2` → 播放顺序 A B A B (重复2次)
  - 支持嵌套反复: `||: A ||: B C :||2 D :||2 E` → 内层先展开, 外层包含内层展开结果
  - 栈结构自动匹配 open/close 配对, 无需预知嵌套深度

- **MIDI事件同步展开**: `midi_converter.py` 的 `convert()` / `convert_all_tracks()` 
  - 新增 `expand_measure_indices()` 静态方法: 输入小节列表 → 输出展开后的索引序列
  - MIDI事件按展开序列生成, 确保音频播放时正确重复对应小节段
  - 不同音轨独立展开(各轨可能有不同的反复结构)

- **时间线同步展开**: `player.py` 的 `build_timeline()`
  - 新增 `_expanded_measure_indices` 缓存属性(由 `rebuild_audio_events()` 计算)
  - 时间线条目按相同展开序列复制: scroll_y 保持不变(视觉位置), time_ms 累加增长
  - 播放光标在反复时会"跳回"之前的谱面位置(符合实际演奏行为)
  - 总时长 `_total_audio_duration_ms` 自动包含反复段的额外时间

### 技术实现

- **展开算法** (`MidiConverter.expand_measure_indices()`):
  ```
  输入: [A, ||:B, C, :||2, D]  (is_repeat_open, repeat_close)
  处理: 
    result=[0], stack=[]
    i=1(B,open): result=[0,1], stack=[1]
    i=2(C): result=[0,1,2]
    i=3(C,close=2): pop→seg=result[1:]=[1,2], 复制1次→result=[0,1,2,1,2]
  输出: [0,1,2,1,2,4]
  ```
  
- **时间线同步**: 
  - 按 global_meas_idx 分组原始条目
  - 按展开序列遍历, running_time 累加推进
  - 每个复制条目: new_time = running_time + (orig_time - base_time)

### 10条Case预期结果

| Case | 场景 | 预期结果 |
| --- | --- | --- |
| 1 | GTP文件无任何反复记号 | 展开序列与原始序列完全一致, 行为与修改前相同 |

## (2026-06-20) - 最近打开文件功能 (独立列表控件)

### 修改内容

- **最近文件列表(独立控件)**: 与文件夹文件列表完全分离的独立 QListWidget，支持折叠/展开
  - 位置: 文件夹文件列表上方，带标题行(标签+展开/折叠按钮)，两列表之间间距 8px
  - **默认折叠**: 只显示1行内容，点击"展开 ▼"按钮显示全部
  - 显示格式: `📄 文件名` (使用强调色/橙色突出显示)
  - 边框样式: 使用强调色边框(与普通文件列表的主色边框区分)
  - 高度自适应: 折叠时1行高度，展开时紧贴所有项数
  - 单击即可快速重新打开(比普通双击更便捷)
  - 鼠标悬停显示完整路径提示
  - 右键菜单: 打开文件 / 在资源管理器中定位 / 从列表移除

- **数据持久化**: 
  - 存储位置: `config/settings.json` 的 `recent_files` 字段
  - 存储内容: 绝对路径列表(最多4条)
  - 自动过滤不存在的文件(启动时清理)

- **自动更新逻辑**:
  - 每次通过 `show_display()` 打开文件时自动记录
  - 新文件插入列表头部(最新的在最前面)
  - 自动去重: 已存在的文件先移除再插入头部
  - 自动截断: 超过4条时删除最旧的记录
  - 仅记录单个文件路径(PDF/GTP/单张图片)，不记录图片列表

### 技术实现

- `_add_recent_file()`: 添加文件到最近列表(去重+截断+持久化+刷新UI)
- `_refresh_recent_list()`: 刷新独立的 recent_list 控件内容
- `_apply_recent_list_height()`: 根据折叠/展开状态设置精确高度
- `_toggle_recent_list()`: 切换展开/折叠状态(翻转布尔值+更新按钮文字+应用高度)
- `_on_recent_file_clicked()`: 最近文件单击事件处理(仅响应recent_list)
- `_show_recent_context_menu()`: 最近文件右键菜单(打开/定位/移除)
- `_remove_recent_file()`: 从最近列表移除指定文件
- `load_config()/save_config()`: 扩展支持 recent_files 字段
- CSS选择器区分: `#recent_list_widget`(强调色) vs `#file_list_widget`(主色)

### UI参数(可调整)

| 参数 | 默认值 | 位置 | 说明 | 调整效果 |
|------|--------|------|------|----------|
| MAX_RECENT_FILES | 4 | SettingsWindow类属性 | 最大保存数量 | 越大占用更多配置空间 |
| _recent_expanded | False | init_ui() | 默认展开/折叠 | True=默认展开，False=默认折叠 |
| 列表间距 | 8px | init_ui() addSpacing | 两列表之间间距 | 可调范围5-10px |
| 按钮高度 | 24px | recent_toggle_btn | 展开/折叠按钮高度 | 增大更易点击 |
