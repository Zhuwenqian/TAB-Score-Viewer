# 功能更新记录

## v1.2.0 (2026-06-07) - Phase 2 增强渲染

### 新增功能

#### 2.1 技巧符号图形化渲染
- **泛音菱形标记**: 自然/人工/拍击/点弦泛音在品格数字外围绘制菱形框（8px大小），视觉区分明显
- **滑音连线**: 滑音(SLIDE_UP/DOWN)自动查找同弦下一个音符并绘制斜线连接，支持跨拍连线
- **推弦弧线箭头**: 推弦(BEND)技巧绘制向上凸起的贝塞尔曲线弧线 + 箭头尖端，弧高18px可调
- **颤音波浪线**: 颤音(VIBRATO)在品格数字上方绘制正弦波浪线(~~~)，振幅2px
- **P.M./Let Ring 虚线延长**: 手制音/延音技巧绘制虚线从起始音符延伸到结束音符，跨多个拍自动连接
- **技巧文字标签**: 所有非图形化技巧仍显示缩写文字（H/P/s/B等），位于品格数字右侧

#### 2.2 时值显示增强
- **附点标记**: 附点音符(is_dotted)在符尾旁绘制4px圆点标记；休止符附点同样支持
- **休止符符号优化**（替代原来的简单圆点）:
  - 全休止符: 悬挂式矩形块(8×6px)在六线谱上方
  - 二分休止符: 坐落式矩形块在六线谱顶部区域
  - 四分休止符: 锯齿形/闪电形折线符号
  - 八分及更短: 类似"7"的圆弧+竖线组合
- **符尾旗美化**: 符尾旗加粗至1.5px、长度增至7px，十六分/三十二分多符尾平行排列更清晰

#### 2.3 调号/拍号/速度标记
- **拍号显示**: 非标准拍号(非4/4)时在每行系统开头显示上下堆叠的分子分母数字（如3/4、6/8）
- **调号显示**: 非C大调/a小调时在拍号左侧显示升降号数量标注（如#1=1个升号, b2=2个降号）
- **兼容处理**: key_signature同时支持int和tuple两种数据格式

### 修改文件
- `gtp_engine/renderer/tab_renderer.py` — 新增7个方法: `_draw_harmonic_diamond`, `_draw_slide_line`, `_draw_bend_indicator`, `_draw_vibrato_wave`, `_draw_pm_letring_extensions`, `_draw_time_signature`, `_draw_key_signature`; 重构 `_draw_technique_marks`, `_draw_beam_flags`, `_draw_rest_symbol`, `_draw_system`, `_draw_beat`, `_draw_measure`

## v1.1.3 (2026-06-06) - 渲染细节优化

### 优化项
- **头部信息仅第1页显示**: 第2页及之后的页面不再绘制标题/调弦/BPM等头部信息，节省约45px垂直空间用于显示更多谱子内容
- **符干/符尾统一向下**: 所有音符的符干方向固定为从第6弦线向下延伸，符尾（尾杠）统一画在六线谱下方，视觉更整洁一致

### 修改文件
- `gtp_engine/renderer/tab_renderer.py` — `_render_page()` 加页码判断; `_draw_stem()` 符干方向改为始终向下

## v1.1.2 (2026-06-06) - 布局引擎重大修复

### Bug 修复

| 问题 | 原因 | 修复方案 |
|------|------|----------|
| **谱子大量空白/只显示一行**（核心问题） | 布局引擎 `_assign_system_coordinates` 为**每个系统**都添加了 `INFO_SECTION_HEIGHT=50px` 的顶部空白，但页面头部信息只在每页绘制一次（绝对坐标 y=15）。6个系统 × 50px = **300px 被浪费为空白**，导致每页只能显示6行而非9行 | 移除每个系统的 `INFO_SECTION_HEIGHT` 偏移量。头部信息使用绝对坐标绘制，不需要在布局中预留空间 |
| **第2页及之后内容挤在页面底部** | 分页时系统的 Y 坐标是连续绝对值（如第2页第1个系统 y_top=1056px），未重置为相对于新页面顶部的坐标，导致内容画在 1200px 页面的底部区域 | 重写 `_split_into_pages()`：分页后对每页的系统重新计算相对 Y 坐标（offset_y = current_y - system.y_top），确保每页内容从顶部开始排布 |

### 渲染参数调整
- **NOTE_MIN_SPACING**: 14 → **22**
  - 效果: 相邻拍之间的水平间距增大 57%，音符不再拥挤
  - 可调范围: 推荐 18~26，越大越宽松但占用更多水平空间

### 修复前后对比 (Demo v5.gp5, 49小节)
| 指标 | 修复前 | 修复后 |
|------|--------|--------|
| 每页系统数 | 6行 + 300px空白 | **9行紧凑排列** |
| 第2页位置 | 内容挤在底部 | **从顶部正常排布** |
| 总页数 | 3页 | 3页（不变，但每页信息密度提升50%） |
| 音符间距 | 14px（拥挤） | **22px（舒适）** |

## v1.1.1 (2026-06-06) - 交互增强与渲染修复

### 新增功能
- **右键菜单"打开文件所在位置"**: 文件列表右键新增"在资源管理器中打开"选项
  - 支持 Windows (`explorer /select,`) 自动定位并高亮选中文件
  - 支持 Linux (xdg-open/nautilus/dolphin/thunar) 多文件管理器兼容
  - 对文件夹和文件均生效
- **GTP音轨选择下拉菜单**: 打开GTP文件后，工具栏显示音轨选择器
  - 下拉列表显示所有音轨：序号 + 名称 + 调弦方案（如 `3. Gui2 (标准调弦EADGBE)`）
  - 切换音轨即时重新渲染对应轨道的六线谱，无需重新加载文件
  - 非GTP文件时自动隐藏，不影响其他格式使用

### Bug 修复

| 问题 | 原因 | 修复方案 |
|------|------|----------|
| 所有音符整体偏移一根弦（往下移了一根琴弦） | PyGuitarPro 的 `note.string` 是 1-based（1~6），但渲染按 0-based 处理，导致每个音符 Y 坐标多算了一个弦间距 | 解析器 `_convert_note()` 中 `string=raw_note.string - 1` 转为 0-based |
| 品格数字显示在线外（底部弦的音符溢出六线区域） | `_draw_note_fret` 中 `text_y = y + ascent//2 - 2` 导致文字基线低于弦线位置 | 改为垂直居中公式 `text_y = y - height//2 + ascent`，文字视觉中心对齐弦线 |
| 调弦匹配使用元组字典键不够健壮 | 虽然当前能工作，但遍历方式对边界情况更安全 | `get_tuning_name()` 改为遍历所有预设调弦逐一比较 |

### 渲染优化
- **小节宽度算法重构**: 从"按时值比例分配空间"改为"固定间距 + 拍数决定宽度"
  - **旧**: 时值短的音符间距小，时值长的间距大（按 duration 比例分配）
  - **新**: 每两个拍之间固定 `NOTE_MIN_SPACING`(14px) 间距，与音符时值无关
  - 效果: 音符密集的小节自然更长(8拍=134px)，稀疏小节更短(6拍=106px)
  - 可调参数: `RenderConfig.NOTE_MIN_SPACING` — 调大则整体更宽松，调小更紧凑

## v1.1.0 (2026-06-06) - GTP六线谱渲染引擎

### 新增功能

#### GTP文件完整渲染（gtp_engine 库）
- **GTP解析引擎**: 基于 PyGuitarPro 库完整解析 .gp3/.gp4/.gp5/.gpx 文件
  - 提取歌曲信息（标题/艺术家/BPM/调号）
  - 解析所有音轨（名称/调弦/乐器类型）
  - 转换为中介数据模型 GTPSong → GTPTrack → GTPMeasure → GTPBeat → GTPNote
- **六线谱渲染引擎**: 使用 QPainter 绘制标准吉他六线谱
  - 6条弦线 + 品格数字（正确显示在对应弦位上）
  - 小节线 + 重复记号（||: / :||）+ 段落标记
  - 符干符尾系统（四分/八分/十六分音符的符干方向和符尾旗）
  - 技巧标记显示（H/P/s/B/P.M./~/N.H./A.H./P.H./tr 等18种技巧缩写）
  - 自动换行布局（按画布宽度自动分行）+ 多页输出
  - 页面头部信息区（标题/轨道名/调弦方案/BPM）
- **可扩展数据模型**: Note/Beat/Measure/Track/Song 全部使用 dataclass，支持任意扩展
- **独立库设计**: gtp_engine 作为独立包位于项目根目录，未来可发布到 PyPI
- **优雅降级**: 当 guitarpro 未安装时自动回退到信息展示图，不影响其他功能

#### 新增依赖
- `guitarpro >= 0.11` (PyGuitarPro) - Guitar Pro 文件解析库（开源项目: pyguitarpro）

## v1.0.0 (2026-06-06) - 初始版本

### 新增功能

#### 核心功能
- **多格式文件支持**: PNG、JPG、JPEG、WEBP 图片格式；PDF 文档渲染；GP3/GP4/GP5/GPX 吉他谱文件预览
- **自动滚动播放**: 谱面自动向上滚动模拟演奏，支持播放/暂停控制
- **速度控制系统**:
  - 基础速度调节（1-500ms），值越小越快
  - 速度曲线编辑器：贝塞尔曲线可视化编辑，支持自定义控制点
  - 预设曲线模板：匀速、渐快、渐慢、先慢后快、先快后慢
- **循环播放功能**:
  - 不循环模式
  - 全局循环模式（从头开始）
  - A-B 区域循环模式（通过进度条 Ctrl+Shift 点击设置）
- **进度条系统**:
  - 自定义 ProgressBarSlider 组件，支持拖动跳转
  - 显示当前位置百分比和已播放时间
  - 可视化标记 A-B 循环区域

#### 标注系统
- **文本标注**: 双击谱面任意位置添加演奏技巧说明文本
- **标注属性**: 支持设置颜色、字体大小、字体族、粗体、背景色
- **标注管理器**: 对话框式批量管理所有标注
  - 新增 / 编辑 / 删除 / 清空操作
  - Ctrl+Z 撤销 / Ctrl+Y 重做（最多50步历史）
  - 取消操作自动恢复状态
- **双击编辑**: 双击已有标注区域直接打开编辑对话框
- **持久化存储**: 标注数据以 JSON 格式保存到 `data/annotations/` 目录

#### 导航与交互
- **页码导航**: PDF / 多图片模式下显示页码输入框，支持直接跳转指定页面
- **鼠标滚轮滚动**:
  - 默认滚动：30px/步
  - Ctrl + 滚轮：快速滚动 100px/步
  - Shift + 滚轮：精细滚动 10px/步
- **键盘快捷键**: 空格播放暂停、方向键调速、ESC 关闭窗口
- **缩放滑块**: 工具栏提供缩放控制（预留接口）

#### UI 设计
- **深色主题**: 现代化暗色界面设计
- **自定义组件**:
  - ModernButton: 圆角按钮，支持 primary/success/accent/danger 四种配色
  - ModernSlider: 深色主题滑块
  - ProgressBarSlider: 带区域标记的进度条
  - SpeedCurveCanvas: 贝塞尔曲线绘制画布
- **响应式布局**: QSplitter 分割布局，可调整谱面与控制面板比例
- **设置页面**: 文件列表浏览界面应用完整深色主题样式

#### 技术架构
- **异步加载**: QThreadPool + QRunnable 多线程加载文件内容
- **信号槽通信**: Worker 通过自定义信号类传递加载结果
- **虚拟环境**: Python venv 隔离运行环境
- **跨平台兼容**: Windows / Linux / Docker 支持，相对路径配置
