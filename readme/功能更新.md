# 功能更新记录

## (2026-06-30) - 主题扩展系统：自定义 JSON/Python 主题 (v2.2.0 / v1.1.5)

### 修改内容

新增主题扩展系统，允许用户通过编写 JSON 或 Python 文件定义自定义主题，查看器和 ApolloTab 自动加载并应用到 UI 与 GTP 渲染。

1. **自动扫描主题目录**: 程序启动时自动扫描项目根目录 `themes/` 下的 `.json` 和 `.py` 文件。
2. **双格式支持**:
   - JSON 文件: 直接包含 `name`/`display_name`/`ui`/`gtp` 字段。
   - Python 文件: 模块级变量 `THEME = {...}`，结构与 JSON 相同，支持动态生成颜色。
3. **UI 与 GTP 同步切换**: 自定义主题同时影响主程序 UI 配色和 ApolloTab 六线谱渲染配色。
4. **安全回退**: 缺失的颜色键自动使用内置深色主题默认值填充；单文件解析失败不影响其他主题加载。
5. **内置主题保护**: 不允许自定义主题覆盖 `dark` 和 `light`。
6. **设置界面联动**: `ThemeManager.available_themes()` 自动包含自定义主题，`SettingsDialog` 主题下拉框实时显示。

**ApolloTab 底层改动**

- `ApolloTab/utils/constants.py`:
  - `ThemeConfig` 新增 `register_theme(name, colors)` 类方法，支持运行时注册自定义主题到 `PRESET_THEMES`
  - 新增 `unregister_theme(name)` 类方法

**主程序与资源改动**

- `TAB Score Viewer.py`:
  - 新增 `CUSTOM_THEMES_DIR` 常量
  - 新增 `_load_theme_from_json()` / `_load_theme_from_py()` / `load_all_custom_themes()`
  - 扩展 `ThemeManager`: 支持注册/注销/切换自定义主题，自动同步到 ApolloTab `ThemeConfig`
  - 程序入口在加载 `settings.json` 前先调用 `load_all_custom_themes()`，保证配置中保存的自定义主题可恢复
- `themes/example_custom.json`: 新增示例主题「护眼纸色 (Paper)」，用户可参照此文件创建自己的主题

### 主题文件格式示例

```json
{
  "name": "my_theme",
  "display_name": "My Theme",
  "ui": {
    "bg_primary": "#121212",
    "text_primary": "#E2E8F0",
    ...
  },
  "gtp": {
    "COLOR_BG": "#1E1E2E",
    "COLOR_TEXT": "#E2E8F0",
    ...
  }
}
```

## (2026-06-30) - 节拍器增强：拍号分母 + 全局增益 (v1.1.4)

### 修改内容

在 v1.1.3 节拍器功能基础上，根据实际使用反馈优化拍号输入与音量表现：

1. **拍号输入改为分数形式**：图片/PDF 模式下，拍号由单一分子输入改为 "分子 / 分母" 两个输入框，支持 4/4、3/4、6/8 等常见拍号，分母范围 1~32。
2. **新增节拍器全局增益**：右侧控制面板增加「增益」滑块（1.0x ~ 5.0x，默认 1.5x），配合通道音量 CC#7=127，解决木鱼音色在伴奏中不够突出、音量滑块开到最大仍被掩盖的问题。
3. **力度曲线调整**：`MetronomeConfig` 基础重拍/普通拍力度调整为 100/70，配合默认增益后最大音量下重拍可达 MIDI 上限 127，普通拍约 105，既提升响度又保留重拍区分度。
4. **GTP 模式同步增益**：增益滑块在 GTP 模式下也保持可见，修改后立即重建 MIDI 事件，无需重启播放。

**修改文件**

| `ApolloTab/audio/metronome.py` | 新增 `gain` 字段；调整 `accent_velocity`/`normal_velocity`；力度缩放乘以 `volume * gain` |
| `ApolloTab/player.py` | `set_metronome()` 增加 `gain` 参数；初始化默认 `gain=2.0` |
| `TAB Score Viewer.py` | 新增拍号分母输入框、节拍器增益滑块及对应回调；更新控件可见性逻辑 |

## (2026-06-29) - 新增节拍器功能 (v1.1.3)

### 修改内容

为吉他谱查看器新增节拍器（Metronome）功能，支持 GTP 模式和图片/PDF 模式：

1. **GTP 模式**: 节拍器自动跟随歌曲 BPM 与拍号，混入 MIDI 事件序列同步播放。
2. **图片/PDF 模式**: 提供手动 BPM（20~300）和拍号分子（1~16）输入，默认 120/4。
3. **UI 控制**: 右侧控制面板新增「节拍器」分组，提供启用开关和音量滑块。
4. **音色选择**: 固定使用 GM 木鱼音色（重拍 High Woodblock 77，普通拍 Low Woodblock 76）。
5. **通道隔离**: 节拍器独占 MIDI 通道 15，避免与旋律轨（0-8/10-14）和鼓轨（9）冲突。

**ApolloTab 底层改动**

- 新增 `ApolloTab/audio/metronome.py`：
  - `MetronomeConfig` 配置类（开关、音量、音高、力度、通道、点击时长）
  - `MetronomeGenerator` 事件生成器，支持 `generate_for_song()` 和 `generate_simple()`
- `ApolloTab/audio/midi_converter.py`：
  - `convert()` / `convert_all_tracks()` 新增 `metronome_config` 参数
  - 启用节拍器时混入节拍器 note_on/note_off 事件
- `ApolloTab/player.py`：
  - 新增 `_metronome_enabled`、`_metronome_volume`、`_metronome_config`
  - 新增 `set_metronome(enabled, volume)` 接口
  - 新增 `play_metronome_only(bpm, numerator, denominator)` 用于图片/PDF 模式
  - `init_audio()` 支持无 song 时初始化（纯节拍器模式）
- `ApolloTab/audio/__init__.py` 与 `ApolloTab/__init__.py`：导出 `MetronomeConfig`、`MetronomeGenerator`

**主程序与资源改动**

- `TAB Score Viewer.py`：新增节拍器 UI 控件、状态保存、事件回调、播放同步
- `locales/zh_CN.json` / `en_US.json` / `ru_RU.json`：新增节拍器相关翻译键

### 核心原理

- 节拍器事件与乐曲事件在同一 MIDI 时间线上按 tick 排序，由 `SynthEngine` 统一播放，保证严格同步。
- 每个点击声持续约 50ms，随后发送 `note_off`。
- 音量通过 `velocity` 线性缩放（0~100 → 0~127）。
- 图片/PDF 模式下，`GTPPlayer` 不加载歌曲，仅初始化 `SynthEngine`，播放时动态生成节拍器事件。

## (2026-06-28) - 修复 GTP 播放音色发闷问题 (v1.1.1)

### 修改内容

修复用户反馈的"播放时好多音都有点闷，但开头正常"问题。根本原因是 MIDI 输出没有正确应用谱面指定的乐器音色，导致部分音轨错误使用了钢琴(program=0)或被播放器强制覆盖为单一清音电吉他(program=27)。

**修复点 1: midi_converter.py 默认音色**
- 旋律轨 `track.instrument == 0` 时，不再发送 `Program Change 0`(大钢琴)
- 改为发送 `Program Change 25`(钢弦吉他，Acoustic Guitar Steel)
- GP 文件明确指定音色时(非0)，仍按原值发送
- 新增类常量 `DEFAULT_MELODY_INSTRUMENT = 25`，便于后续统一调整

**修复点 2: player.py 移除强制音色覆盖**
- 删除 `rebuild_audio_events()` 中"为每个通道设置合适的乐器音色"的循环
- 不再调用 `set_instrument(ch, 27)` 把所有非鼓通道强制设为清音电吉他
- 音色完全由 `MidiConverter` 生成的 `Bank Select + Program Change` 事件决定
- 保留 `convert_all_tracks` 原有音色事件生成逻辑，GP7/GP8 解析的 bank 和 program 可正确生效

### 核心原理

1. **GM 音色号**: 0=Grand Piano(钢琴，延音长、听起来闷), 25=Acoustic Guitar Steel(钢弦吉他)
2. **事件时序**: `Bank Select MSB(CC#0) → Bank Select LSB(CC#32) → Program Change` 仍保持原有排序，确保 bank/program 正确切换
3. **播放状态一致性**: 移除 player 层的强制覆盖后，`seek()` 到中后段时不会再因跳过 `tick=0` 的 program_change 而错用 27 号音色

| `ApolloTab/player.py` | 删除 `rebuild_audio_events()` 中强制 `set_instrument(ch, 27)` 的循环，音色由 midi_converter 统一处理 |
